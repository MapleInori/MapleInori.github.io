在面向对象编程（OOP）中，继承是实现多态性的核心机制之一。C# 通过 `<span class="ne-text">virtual</span>`、`<span class="ne-text">override</span>` 和 `<span class="ne-text">new</span>` 关键字提供了两种实现多态的重要方式：**重写（Overriding）** 和  **隐藏（Hiding）** **。理解它们的区别对于在 Unity 中构建灵活、可维护的代码架构至关重要。**

## 什么是虚方法

**虚方法（Virtual Method）** 是在父类中使用 `<span class="ne-text">virtual</span>` 关键字声明的方法。它的目的是明确表示该方法**期望**被子类以某种方式重新定义或改变其行为。

* **声明语法** ： 在方法返回类型前加上 **`<span class="ne-text">virtual</span>` 关键字。
* **目的** **： 为后续的继承者提供一个可以自定义的“模板”或“钩子”，是实现运行时多态**的基础。

```csharp
public class Enemy : MonoBehaviour
{
    // 声明一个虚方法
    public virtual void TakeDamage(int damageAmount)
    {
        Debug.Log("Enemy took " + damageAmount + " damage.");
        // ... 扣血等通用逻辑
    }
}
```

## 重写虚方法

**重写（Overriding）** 是子类使用 `<span class="ne-text">override</span>` 关键字为继承的虚方法提供全新实现的过程。这是实现多态最标准、最推荐的方式。

* **声明语法** **： 在子类方法返回类型前使用 **`<span class="ne-text">override</span>` 关键字。
* **核心特性** **： 无论通过父类引用还是子类引用调用该方法，最终执行的都将是子类的重写版本。这是在****运行时**根据对象的实际类型决定的。

```csharp
public class Boss : Enemy // 继承自 Enemy
{
    // 重写父类的虚方法
    public override void TakeDamage(int damageAmount)
    {
        // 可以先调用父类的通用逻辑（可选）
        // base.TakeDamage(damageAmount);

        // 然后提供Boss特有的逻辑
        Debug.Log("Boss roared and took reduced damage: " + (damageAmount / 2));
        // ... Boss特有的扣血逻辑（例如减半伤害）
    }
}
```

## 隐藏虚方法

 **隐藏（Hiding）** ，也称为“使用 **`<span class="ne-text">new</span>` 关键字进行隐藏”，是子类中声明一个与父类方法同名的方法（无论父类方法是否为虚方法），并使用 `<span class="ne-text">new</span>` 关键字显式表明意图。

* **声明语法** ： 在子类方法返回类型前使用 **`<span class="ne-text">new</span>` 关键字（可省略，但省略会引发编译器警告）。
* **核心特性** **： 它并不会重写父类的方法，而是在子类中创建了一个** **全新的、独立的方法** **，该方法只是恰好与父类方法同名。调用哪个版本取决于编译时**的引用类型，而不是运行时的对象类型。

```csharp
public class StealthEnemy : Enemy // 继承自 Enemy
{
    // 使用 new 关键字隐藏父类的 TakeDamage 方法
    public new void TakeDamage(int damageAmount)
    {
        Debug.Log("Stealth Enemy avoided the attack!");
        // ... 闪避、隐身等特有逻辑，完全取代父类逻辑
    }
}
```

## 不同情况下的调用对比

**让我们通过代码示例来直观感受 **`<span class="ne-text">override</span>` 和 `<span class="ne-text">new</span>` 在不同调用场景下的巨大差异。
假设我们有以下类结构：

```csharp
public class Enemy {
    public virtual void Log() { Debug.Log("I am an Enemy."); }
}

public class Boss : Enemy {
    public override void Log() { Debug.Log("I am a Boss!"); } // 重写
}

public class StealthEnemy : Enemy {
    public new void Log() { Debug.Log("I am a StealthEnemy!"); } // 隐藏
}
```

### 通过子类引用调用

```csharp
Boss boss = new Boss();
StealthEnemy stealth = new StealthEnemy();

boss.Log();    // 输出: "I am a Boss!" (调用重写版本)
stealth.Log(); // 输出: "I am a StealthEnemy!" (调用隐藏的新方法)
```

### 通过父类引用调用（子类实例）

**这是最关键的区别所在，也是多态性的体现之处。**

```csharp
Enemy enemy1 = new Boss();         // 父类引用指向Boss实例
Enemy enemy2 = new StealthEnemy(); // 父类引用指向StealthEnemy实例

enemy1.Log(); // 输出: "I am a Boss!" (运行时查找，调用重写版本)
enemy2.Log(); // 输出: "I am an Enemy." (编译时决定，调用父类原始版本)
```

 **结论** **：**

* `<span class="ne-text">override</span>`：  **运行时多态** **。**`<span class="ne-text">enemy1</span>` 的引用类型是 `<span class="ne-text">Enemy</span>`，但实际对象是 `<span class="ne-text">Boss</span>`，因此调用 `<span class="ne-text">Boss</span>` 的重写方法。
* `<span class="ne-text">new</span>`：  **编译时绑定** **。**`<span class="ne-text">enemy2</span>` 的引用类型是 `<span class="ne-text">Enemy</span>`，编译器看到 `<span class="ne-text">Enemy.Log()</span>` 是虚方法，但它发现 `<span class="ne-text">StealthEnemy</span>` 没有重写它，只是隐藏了它，因此直接绑定到 `<span class="ne-text">Enemy</span>` 类的原始方法。

### 使用强制类型转换

```csharp
Enemy enemyRef = new StealthEnemy();

enemyRef.Log();                    // 输出: "I am an Enemy." (父类版本)
((StealthEnemy)enemyRef).Log();    // 输出: "I am a StealthEnemy!" (子类新方法)
```

 **结论** **： 将父类引用强制转换回具体的子类类型后，可以访问到那个被“隐藏”的独立方法。**

### 多级继承中的隐藏

**隐藏可以在任何层级发生，行为规则一致。**

```csharp
public class SuperStealthEnemy : StealthEnemy {
    public new void Log() { Debug.Log("I am Super Stealthy!"); }
}

Enemy e = new SuperStealthEnemy();
StealthEnemy s = new SuperStealthEnemy();

// 一看是虚方法，往下看有没有重写，子类重写则调用子类方法，子类隐藏则向上找非隐藏方法
e.Log(); // 输出: "I am an Enemy." (追溯到第一个非隐藏的虚方法)
s.Log(); // 输出: "I am a StealthEnemy!" (编译时根据s的类型StealthEnemy决定)
((SuperStealthEnemy)e).Log(); // 输出: "I am Super Stealthy!"
```

## 虚方法与非虚方法的调用对比

**这种情况虽然复杂且不推荐，但有助于理解其机制。**

```csharp
public class A { public virtual void Method() { Debug.Log("A"); } }
public class B : A { public override void Method() { Debug.Log("B"); } }
public class C : B { public new virtual void Method() { Debug.Log("C"); } } // 隐藏B的方法，并把自己变成新的虚方法
public class D : C { public override void Method() { Debug.Log("D"); } } // 重写C的新虚方法

A a = new D();
a.Method(); // 输出: "B"
// 解释：a的编译类型是A。它查找A的虚方法Method。
// D 的继承链：A.virtual -> B.override -> C.new (这中断了重写链，创建了新方法) -> D.override (重写的是C的方法，不是A的)
// 因此，对于A的虚方法表，最后的重写是B完成的。

C c = new D();
c.Method(); // 输出: "D"
// 解释：c的编译类型是C。C.Method()是新的虚方法，并且被D重写了。
```
