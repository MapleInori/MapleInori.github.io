## 什么是安全区域

全面屏手机、刘海屏、挖孔屏、底部手势条，都会让屏幕中有一部分区域不适合直接放按钮、文本和重要 UI。
Unity 提供了 `Screen.safeArea`，它表示当前设备上真正适合放内容的矩形区域。

如果你的 UI 直接铺满全屏，而没有做安全区域适配，就很容易出现这些问题：

- 顶部标题被刘海挡住
- 底部按钮压到手势条
- 左右异形屏把边缘按钮裁掉
- 横屏时布局突然错位

核心思路是：

1. 获取当前平台的安全区域
2. 把安全区域从像素坐标换算成锚点坐标
3. 把结果应用到某个 `RectTransform`
4. 在屏幕尺寸、方向、安全区变化时自动刷新

## 先说结论

如果你只是普通的 Unity App，很多时候直接用 `Screen.safeArea` 就够了。
如果你接入了微信小游戏、抖音小游戏这类平台 SDK，经常还需要自己把平台返回的安全区域坐标转换成 Unity 使用的 `Rect`。

另外，安全区域适配通常只应该作用在“内容层”，而不是整个背景层。

推荐层级：

```text
Canvas
|- FullScreenBackground
|- SafeAreaRoot
   |- TopBar
   |- Content
   |- BottomBar
```

这样背景可以继续全屏铺满，而真正需要点击和阅读的 UI 内容放进安全区域内。

## 整体脚本结构

我们先看这个组件要负责什么：

- 挂在一个 UI 顶层节点上
- 自动获取 `RectTransform`
- 持续检测屏幕尺寸和安全区变化
- 根据开关决定是否适配左右、顶部、底部
- 最终把结果写入锚点

下面按代码块拆开讲。

## 第一步：定义字段和开关

这一步的重点不是“写代码”，而是把需求抽象清楚。

```csharp
using UnityEngine; // Unity 核心命名空间
using WeChatWASM; // 微信小游戏 SDK 命名空间

namespace MapleSnowFramework.UI
{
    /// <summary>
    /// 安全区域适配组件。
    /// 用于把当前节点限制在设备可安全显示的 UI 区域内。
    /// </summary>
    public class SafeArea : MonoBehaviour
    {
        private RectTransform panel; // 当前组件所在节点的矩形变换组件
        private Rect lastSafeArea = new Rect(0, 0, 0, 0); // 上一次应用过的安全区域
        private Vector2Int lastScreenSize = Vector2Int.zero; // 上一次记录的屏幕尺寸
        private ScreenOrientation lastOrientation = ScreenOrientation.AutoRotation; // 上一次记录的屏幕方向

        [SerializeField] private bool conformX = true; // 是否适配左右安全区
        [SerializeField] private bool conformY = true; // 是否启用上下安全区适配
        [SerializeField] private bool conformTop = true; // 是否适配顶部安全区
        [SerializeField] private bool conformBottom = true; // 是否适配底部安全区
        [SerializeField] private bool logging = false; // 是否输出调试日志
    }
}
```

这些字段分别控制什么：

- `panel`：当前脚本所在物体的 `RectTransform`
- `lastSafeArea`：记录上一次应用过的安全区域，避免重复刷新
- `lastScreenSize`：记录上一次屏幕尺寸
- `lastOrientation`：记录上一次屏幕方向
- `conformX`：是否适配左右安全区
- `conformY`：是否启用纵向安全区适配
- `conformTop`：是否只保留顶部安全区
- `conformBottom`：是否只保留底部安全区
- `logging`：调试时打印日志

这里把顶部和底部分开控制，是因为实际项目里很常见这几种情况：

- 背景图希望铺满屏幕，但顶部标题不能进刘海
- 底部操作栏要避开手势条，但顶部背景不需要缩进
- 横屏项目只想处理左右，不想处理上下

## 第二步：初始化组件并监听变化

安全区域不是一成不变的。横竖屏切换、分辨率变化、某些平台窗口变化，都会让它发生变化，所以不能只在 `Start` 里算一次。

```csharp
/// <summary>
/// 在组件初始化时获取依赖并立即执行一次安全区刷新。
/// </summary>
private void Awake()
{
    panel = GetComponent<RectTransform>(); // 获取当前节点的 RectTransform

    if (panel == null)
    {
        enabled = false; // 如果没有 RectTransform，则禁用组件
        return; // 直接结束，避免后续空引用
    }

    Refresh(); // 初始化时先应用一次安全区域
}

/// <summary>
/// 每帧检测屏幕状态是否变化，并在需要时重新应用安全区域。
/// </summary>
private void Update()
{
    Refresh(); // 持续检查安全区域是否需要刷新
}

/// <summary>
/// 当安全区域、屏幕尺寸或方向发生变化时，重新应用布局。
/// </summary>
private void Refresh()
{
    Rect safeArea = GetSafeArea(); // 获取当前平台的安全区域

    if (safeArea != lastSafeArea
        || Screen.width != lastScreenSize.x
        || Screen.height != lastScreenSize.y
        || Screen.orientation != lastOrientation)
    {
        lastSafeArea = safeArea; // 缓存本次安全区域
        lastScreenSize = new Vector2Int(Screen.width, Screen.height); // 缓存本次屏幕尺寸
        lastOrientation = Screen.orientation; // 缓存本次屏幕方向

        ApplySafeArea(safeArea); // 把安全区域应用到当前面板
    }
}
```

这里有两个实现重点。

第一，`Awake()` 里先拿 `RectTransform`。
这个组件本质上就是给 UI 用的，如果挂到一个没有 `RectTransform` 的普通物体上，它本来就不该继续执行。

第二，虽然 `Update()` 每帧都会调用 `Refresh()`，但真正应用锚点时做了变化检测，所以不会每帧都重复改布局，开销很小。

## 第三步：普通 Unity 平台直接取 `Screen.safeArea`

如果你现在做的是普通 Unity App、编辑器测试，最简单的安全区获取方式就是下面这样：

```csharp
/// <summary>
/// 获取 Unity 原生提供的安全区域。
/// </summary>
private Rect GetSafeArea()
{
    return Screen.safeArea; // 直接返回 Unity 计算好的安全区域
}
```

这一段简单，但它只适合 Unity 自己已经正确处理了安全区的平台。

问题出在小游戏 SDK：
有些平台返回的安全区域不是 Unity 的坐标系，而是平台自己的坐标系，所以要先转换。

## 第四步：微信小游戏安全区坐标转换

这份草稿的主体实现是微信小游戏版本，所以正文主线也以它为例。

微信返回的安全区通常有两个特点：

- 原点在左上角
- 宽高和边界值往往是逻辑像素，需要乘上 `pixelRatio`

而 Unity 的 `Rect` 使用的是左下角原点，所以 `y` 必须转换。

```csharp
/// <summary>
/// 获取微信小游戏平台的安全区域，并转换成 Unity 使用的坐标系。
/// </summary>
private Rect GetSafeArea()
{
    var wxInfo = WX.GetWindowInfo(); // 获取微信小游戏窗口信息

    if (wxInfo.safeArea == null)
    {
        return new Rect(0, 0, Screen.width, Screen.height); // 如果平台没有返回安全区，则退回全屏
    }

    var wxSafeArea = wxInfo.safeArea; // 读取微信平台返回的安全区域
    float pixelRatio = (float)wxInfo.pixelRatio; // 读取设备像素比

    float x = (float)((wxSafeArea.right - wxSafeArea.width) * pixelRatio); // 由右边界和宽度反推左边界
    float y = (float)((wxInfo.screenHeight - wxSafeArea.bottom) * pixelRatio); // 把左上原点坐标转换成左下原点坐标
    float width = (float)(wxSafeArea.width * pixelRatio); // 宽度从逻辑像素换算为物理像素
    float height = (float)(wxSafeArea.height * pixelRatio); // 高度从逻辑像素换算为物理像素

    return new Rect(x, y, width, height); // 返回 Unity 可直接使用的安全区域
}
```

这四个值要这样理解：

`x`

```csharp
float x = (float)((wxSafeArea.right - wxSafeArea.width) * pixelRatio); // 通过右边界减宽度得到左边界
```

微信给了安全区右边界和宽度，所以左边界就是：

```text
left = right - width
```

Unity 的 `Rect.x` 需要的就是左边界，因此这样换算。

`y`

```csharp
float y = (float)((wxInfo.screenHeight - wxSafeArea.bottom) * pixelRatio); // 把左上原点的底边坐标转换为 Unity 的左下原点坐标
```

微信的原点在左上，`bottom` 表示“从顶部往下数到安全区底边”的距离。
Unity 的原点在左下，所以要用：

```text
UnityY = 屏幕总高 - 微信安全区底边
```

这样就把坐标翻过来了。

`width` 和 `height`

```csharp
float width = (float)(wxSafeArea.width * pixelRatio); // 把逻辑像素宽度换算成物理像素宽度
float height = (float)(wxSafeArea.height * pixelRatio); // 把逻辑像素高度换算成物理像素高度
```

平台给的是逻辑像素，而 Unity UI 实际布局一般按物理像素工作，所以这里要乘 `pixelRatio`。

这里还有一个很重要的兜底：

```csharp
if (wxInfo.safeArea == null)
{
    return new Rect(0, 0, Screen.width, Screen.height); // 没有安全区域数据时使用整屏作为兜底
}
```

不是所有机型、所有平台实现都会稳定返回 `safeArea`。
拿不到时直接退回全屏，至少不会让 UI 完全跑飞。

## 第五步：如果是抖音小游戏，换成对应坐标转换

如果你的项目接的是抖音小游戏 SDK，可以把获取安全区的方法换成下面这一版：

```csharp
/// <summary>
/// 根据不同平台返回的安全区域数据，生成 Unity 可用的 Rect。
/// </summary>
private Rect GetSafeArea()
{
#if UNITY_EDITOR // 编辑器环境直接使用 Unity 原生安全区域
    return Screen.safeArea; // 返回编辑器下的安全区域
#elif TTSDK_MIX_ENGINE // 抖音小游戏环境
    TTSDK.TTSystemInfo ttInfo = TTSDK.TT.GetSystemInfo(); // 获取抖音平台系统信息
    TTSDK.SafeArea ttSafeArea = ttInfo.safeArea; // 获取抖音平台安全区域

    float x = ttSafeArea.right - ttSafeArea.width; // 由右边界和宽度换算左边界
    float y = ttInfo.screenHeight - ttSafeArea.bottom; // 把左上原点的坐标转换为左下原点
    float width = ttSafeArea.width; // 读取安全区宽度
    float height = ttSafeArea.height; // 读取安全区高度

    float pixelRatio = (float)ttInfo.pixelRatio; // 读取设备像素比
    return new Rect(x * pixelRatio, y * pixelRatio, width * pixelRatio, height * pixelRatio); // 转为 Unity 使用的物理像素 Rect
#else // 其他平台
    return Screen.safeArea; // 默认走 Unity 原生安全区域
#endif
}
```

和微信版本本质上一样，区别只在于 SDK 类型和字段来源不同。
真正要记住的是这一条：

```text
平台坐标系 != Unity 坐标系 时，要先做坐标转换，再把结果交给 UI
```

## 第六步：把安全区应用到 RectTransform

拿到安全区以后，不要直接改 `sizeDelta` 或硬算位置。
最稳的方式是把安全区换算成锚点，然后作用到 `anchorMin` 和 `anchorMax`。

```csharp
/// <summary>
/// 把计算好的安全区域转换成锚点并应用到当前 RectTransform。
/// </summary>
private void ApplySafeArea(Rect safeArea)
{
    Rect finalArea = safeArea; // 复制一份原始安全区域，后续按开关修改

    if (!conformX)
    {
        finalArea.x = 0; // 不适配左右时，从屏幕最左边开始
        finalArea.width = Screen.width; // 不适配左右时，宽度拉满全屏
    }

    if (conformY)
    {
        float screenHeight = Screen.height; // 记录当前屏幕高度
        float safeBottom = safeArea.y; // 安全区底边离屏幕底部的距离
        float safeTop = safeArea.y + safeArea.height; // 安全区顶边离屏幕底部的距离

        if (conformTop && !conformBottom)
        {
            finalArea.y = 0; // 底边贴齐屏幕底部
            finalArea.height = safeTop; // 高度保留到安全区顶部
        }
        else if (!conformTop && conformBottom)
        {
            finalArea.y = safeBottom; // 底部避开手势条
            finalArea.height = screenHeight - safeBottom; // 顶部继续铺满屏幕
        }
        else if (!conformTop && !conformBottom)
        {
            finalArea.y = 0; // 上下都不适配时贴到底部
            finalArea.height = screenHeight; // 上下都不适配时高度拉满
        }
    }
    else
    {
        finalArea.y = 0; // 关闭纵向适配时贴到底部
        finalArea.height = Screen.height; // 关闭纵向适配时高度拉满
    }

    Vector2 anchorMin = new Vector2(
        Mathf.Clamp01(finalArea.x / Screen.width), // 计算左锚点的 X 比例
        Mathf.Clamp01(finalArea.y / Screen.height) // 计算下锚点的 Y 比例
    );

    Vector2 anchorMax = new Vector2(
        Mathf.Clamp01((finalArea.x + finalArea.width) / Screen.width), // 计算右锚点的 X 比例
        Mathf.Clamp01((finalArea.y + finalArea.height) / Screen.height) // 计算上锚点的 Y 比例
    );

    if (float.IsNaN(anchorMin.x) || float.IsNaN(anchorMin.y)
        || float.IsNaN(anchorMax.x) || float.IsNaN(anchorMax.y))
    {
        return; // 如果出现非法数值，则放弃本次应用
    }

    panel.anchorMin = anchorMin; // 设置左下锚点
    panel.anchorMax = anchorMax; // 设置右上锚点
    panel.offsetMin = Vector2.zero; // 清空左下偏移
    panel.offsetMax = Vector2.zero; // 清空右上偏移

    if (logging)
    {
        Debug.LogFormat(
            "[SafeArea] {0} => x:{1}, y:{2}, w:{3}, h:{4}",
            name, // 当前对象名称
            finalArea.x, // 最终区域的左边界
            finalArea.y, // 最终区域的底边界
            finalArea.width, // 最终区域的宽度
            finalArea.height // 最终区域的高度
        ); // 输出调试日志
    }
}
```

这一段是整个脚本最核心的部分。

### 1. 先得到最终要用的区域

```csharp
Rect finalArea = safeArea; // 复制一份安全区域，方便后续按规则修改
```

先把原始安全区复制出来，后面所有开关都改 `finalArea`，不要直接改原值，这样逻辑最清楚。

### 2. 控制是否适配左右

```csharp
if (!conformX)
{
    finalArea.x = 0; // 左边界恢复为整屏最左侧
    finalArea.width = Screen.width; // 宽度扩展为整屏宽度
}
```

如果你不想处理左右异形区，就直接把宽度拉成全屏。

### 3. 控制是否适配上下

```csharp
if (conformY)
{
    float screenHeight = Screen.height; // 当前屏幕总高度
    float safeBottom = safeArea.y; // 安全区底边高度
    float safeTop = safeArea.y + safeArea.height; // 安全区顶边高度
```

这里先拆出两个关键值：

- `safeBottom`：安全区底边离屏幕底部的距离
- `safeTop`：安全区顶边离屏幕底部的距离

后面的四种情况都是围绕这两个值做。

#### 情况一：只适配顶部

```csharp
if (conformTop && !conformBottom)
{
    finalArea.y = 0; // 底部拉到整屏最下方
    finalArea.height = safeTop; // 顶部仍然遵守安全区
}
```

意思是：

- 顶部别进刘海
- 底部可以一直铺到屏幕最下方

这个很适合顶部标题栏、顶部按钮区域。

#### 情况二：只适配底部

```csharp
else if (!conformTop && conformBottom)
{
    finalArea.y = safeBottom; // 底部遵守安全区
    finalArea.height = screenHeight - safeBottom; // 顶部继续铺满
}
```

意思是：

- 底部避开手势条
- 顶部继续铺满

这个很适合底部操作栏。

#### 情况三：顶部和底部都适配

```csharp
else if (conformTop && conformBottom)
{
    // 保持原始安全区，上下都遵守系统返回的安全区域
}
```

这是最标准的“整块内容全部进安全区域”。

#### 情况四：上下都不适配

```csharp
else if (!conformTop && !conformBottom)
{
    finalArea.y = 0; // 底边恢复为整屏底部
    finalArea.height = screenHeight; // 高度恢复为整屏高度
}
```

这相当于纵向全屏。

### 4. 把像素坐标换成锚点

```csharp
Vector2 anchorMin = new Vector2(
    Mathf.Clamp01(finalArea.x / Screen.width), // 左边界换算成 0 到 1 的比例
    Mathf.Clamp01(finalArea.y / Screen.height) // 下边界换算成 0 到 1 的比例
);

Vector2 anchorMax = new Vector2(
    Mathf.Clamp01((finalArea.x + finalArea.width) / Screen.width), // 右边界换算成 0 到 1 的比例
    Mathf.Clamp01((finalArea.y + finalArea.height) / Screen.height) // 上边界换算成 0 到 1 的比例
);
```

`RectTransform.anchorMin` 和 `anchorMax` 取值范围是 `0~1`，所以这里要做归一化。

例如：

- 左边界在屏幕中点，`anchorMin.x = 0.5`
- 顶边在屏幕最上方，`anchorMax.y = 1`

使用锚点而不是写死像素值的好处是，分辨率变化时 UI 仍然能保持正确比例。

### 5. 为什么还要把 offset 清零

```csharp
panel.offsetMin = Vector2.zero; // 清空左下偏移量
panel.offsetMax = Vector2.zero; // 清空右上偏移量
```

这一句很重要。

如果你只改锚点，不清 `offsetMin / offsetMax`，原本 Inspector 里残留的偏移值还会继续生效，最后就可能出现“明明安全区算对了，但 UI 还是偏了一点”的问题。

所以最稳妥的做法是：

- 锚点负责决定安全区范围
- 偏移量全部清零

## 第七步：完整脚本

如果你想直接复制使用，下面是一份整理后的完整版本。
这里保留的是草稿里的微信小游戏实现，适合已经接入 `WeChatWASM` 的项目。

```csharp
using UnityEngine; // Unity 核心命名空间
using WeChatWASM; // 微信小游戏 SDK 命名空间

namespace MapleSnowFramework.UI
{
    /// <summary>
    /// 用于异形屏设备的安全区域适配。
    /// 挂到需要进入安全区的 UI 顶层节点上即可。
    /// </summary>
    public class SafeArea : MonoBehaviour
    {
        private RectTransform panel; // 当前节点的 RectTransform
        private Rect lastSafeArea = new Rect(0, 0, 0, 0); // 上一次应用的安全区域
        private Vector2Int lastScreenSize = Vector2Int.zero; // 上一次记录的屏幕尺寸
        private ScreenOrientation lastOrientation = ScreenOrientation.AutoRotation; // 上一次记录的屏幕方向

        [SerializeField] private bool conformX = true; // 是否适配左右安全区
        [SerializeField] private bool conformY = true; // 是否适配上下安全区
        [SerializeField] private bool conformTop = true; // 是否保留顶部安全区
        [SerializeField] private bool conformBottom = true; // 是否保留底部安全区
        [SerializeField] private bool logging = false; // 是否打印调试日志

        /// <summary>
        /// 在组件初始化时获取 RectTransform，并立即应用一次安全区域。
        /// </summary>
        private void Awake()
        {
            panel = GetComponent<RectTransform>(); // 获取当前对象的 RectTransform

            if (panel == null)
            {
                enabled = false; // 如果没有 RectTransform，则禁用组件
                return; // 提前结束，避免后续空引用
            }

            Refresh(); // 初始化时立即刷新一次安全区域
        }

        /// <summary>
        /// 每帧检测屏幕状态，必要时重新适配安全区域。
        /// </summary>
        private void Update()
        {
            Refresh(); // 持续检查是否需要重新应用安全区域
        }

        /// <summary>
        /// 检查安全区域、屏幕尺寸和方向是否发生变化。
        /// </summary>
        private void Refresh()
        {
            Rect safeArea = GetSafeArea(); // 获取当前平台的安全区域

            if (safeArea != lastSafeArea
                || Screen.width != lastScreenSize.x
                || Screen.height != lastScreenSize.y
                || Screen.orientation != lastOrientation)
            {
                lastSafeArea = safeArea; // 缓存当前安全区域
                lastScreenSize = new Vector2Int(Screen.width, Screen.height); // 缓存当前屏幕尺寸
                lastOrientation = Screen.orientation; // 缓存当前屏幕方向

                ApplySafeArea(safeArea); // 重新应用安全区域
            }
        }

        /// <summary>
        /// 获取当前平台的安全区域。
        /// 编辑器走 Unity 原生接口，微信小游戏走 SDK 坐标转换。
        /// </summary>
        private Rect GetSafeArea()
        {
#if UNITY_EDITOR // 编辑器环境
            return Screen.safeArea; // 直接使用 Unity 原生安全区域
#else // 微信小游戏环境
            var wxInfo = WX.GetWindowInfo(); // 获取微信小游戏窗口信息

            if (wxInfo.safeArea == null)
            {
                return new Rect(0, 0, Screen.width, Screen.height); // 如果没有安全区数据，则退回整屏
            }

            var wxSafeArea = wxInfo.safeArea; // 读取微信平台返回的安全区域
            float pixelRatio = (float)wxInfo.pixelRatio; // 读取像素比

            float x = (float)((wxSafeArea.right - wxSafeArea.width) * pixelRatio); // 计算左边界并转为物理像素
            float y = (float)((wxInfo.screenHeight - wxSafeArea.bottom) * pixelRatio); // 把左上原点转换成左下原点
            float width = (float)(wxSafeArea.width * pixelRatio); // 宽度转为物理像素
            float height = (float)(wxSafeArea.height * pixelRatio); // 高度转为物理像素

            return new Rect(x, y, width, height); // 返回 Unity 可直接使用的安全区域
#endif
        }

        /// <summary>
        /// 把安全区域转换成锚点，并应用到当前 RectTransform。
        /// </summary>
        private void ApplySafeArea(Rect safeArea)
        {
            Rect finalArea = safeArea; // 复制一份安全区域用于后续修改

            if (!conformX)
            {
                finalArea.x = 0; // 不适配左右时从屏幕最左边开始
                finalArea.width = Screen.width; // 不适配左右时宽度拉满
            }

            if (conformY)
            {
                float screenHeight = Screen.height; // 当前屏幕总高度
                float safeBottom = safeArea.y; // 安全区底边距离屏幕底部的距离
                float safeTop = safeArea.y + safeArea.height; // 安全区顶边距离屏幕底部的距离

                if (conformTop && !conformBottom)
                {
                    finalArea.y = 0; // 底部铺满到屏幕底边
                    finalArea.height = safeTop; // 顶部仍然遵守安全区
                }
                else if (!conformTop && conformBottom)
                {
                    finalArea.y = safeBottom; // 底部保留安全区
                    finalArea.height = screenHeight - safeBottom; // 顶部继续铺满
                }
                else if (!conformTop && !conformBottom)
                {
                    finalArea.y = 0; // 上下都不适配时从底边开始
                    finalArea.height = screenHeight; // 上下都不适配时高度拉满
                }
            }
            else
            {
                finalArea.y = 0; // 关闭纵向适配时从底边开始
                finalArea.height = Screen.height; // 关闭纵向适配时高度拉满
            }

            Vector2 anchorMin = new Vector2(
                Mathf.Clamp01(finalArea.x / Screen.width), // 左边界归一化为锚点 X
                Mathf.Clamp01(finalArea.y / Screen.height) // 下边界归一化为锚点 Y
            );

            Vector2 anchorMax = new Vector2(
                Mathf.Clamp01((finalArea.x + finalArea.width) / Screen.width), // 右边界归一化为锚点 X
                Mathf.Clamp01((finalArea.y + finalArea.height) / Screen.height) // 上边界归一化为锚点 Y
            );

            if (float.IsNaN(anchorMin.x) || float.IsNaN(anchorMin.y)
                || float.IsNaN(anchorMax.x) || float.IsNaN(anchorMax.y))
            {
                return; // 如果锚点计算结果非法，则跳过这次应用
            }

            panel.anchorMin = anchorMin; // 设置左下锚点
            panel.anchorMax = anchorMax; // 设置右上锚点
            panel.offsetMin = Vector2.zero; // 清空左下偏移量
            panel.offsetMax = Vector2.zero; // 清空右上偏移量

            if (logging)
            {
                Debug.LogFormat(
                    "[SafeArea] {0} => x:{1}, y:{2}, w:{3}, h:{4}",
                    name, // 当前对象名称
                    finalArea.x, // 最终区域左边界
                    finalArea.y, // 最终区域底边界
                    finalArea.width, // 最终区域宽度
                    finalArea.height // 最终区域高度
                ); // 输出调试日志
            }
        }
    }
}
```

如果你不是微信小游戏项目：

- 普通 Unity 项目：把 `GetSafeArea()` 换成 `return Screen.safeArea;`
- 抖音小游戏项目：把 `GetSafeArea()` 换成前面那段 TTSDK 版本
- 没有接微信 SDK 的工程：删掉 `using WeChatWASM;`

## 挂载和使用方法

最常见的使用方式有两种。

### 用法一：整个内容面板都进入安全区域

```text
Canvas
|- SafeAreaRoot
   |- TopBar
   |- MainContent
   |- BottomBar
```

把 `SafeArea` 脚本挂在 `SafeAreaRoot` 上。
这样整个内容区都会缩进到安全区域内。

### 用法二：背景全屏，内容进安全区域

```text
Canvas
|- Panel
   |- Background
   |- SafeAreaRoot
      |- Title
      |- Buttons
      |- Notice
```

这也是最推荐的方式。

原因很简单：背景图通常希望铺满整屏，看起来更自然

如果你把背景也一起缩进安全区，画面边缘可能会露空，看起来像“画面没铺满”。

## 开关怎么配

下面这张表可以直接当使用说明。

| 配置                                          | 效果               |
| --------------------------------------------- | ------------------ |
| `conformX = true`                           | 左右也进入安全区域 |
| `conformX = false`                          | 左右继续全屏       |
| `conformY = true`                           | 启用上下安全区逻辑 |
| `conformY = false`                          | 上下直接全屏       |
| `conformTop = true, conformBottom = true`   | 顶部和底部都避让   |
| `conformTop = true, conformBottom = false`  | 只避让顶部         |
| `conformTop = false, conformBottom = true`  | 只避让底部         |
| `conformTop = false, conformBottom = false` | 上下都不避让       |

## 常见坑

### 1. 安全区域不是分辨率适配的替代品

安全区域解决的是“别被异形屏遮住”。
Canvas Scaler 解决的是“不同分辨率下 UI 比例怎么变”。

这两个问题不是一回事，项目里通常要同时使用。

### 2. 只改锚点，不清 offset

这是最容易漏掉的点之一。
不清 `offsetMin / offsetMax`，很容易出现边缘多出几像素偏移的问题。

### 3. 把背景也一起塞进安全区

有时这不是 bug，而是结构设计不合理。
背景全屏、内容进安全区，一般会比“整个面板一起缩”更好看。

### 4. 平台坐标原点不一样

Unity 是左下角原点。
小游戏 SDK 很多是左上角原点。

如果你拿到平台安全区后直接套进 Unity，不做 `y` 轴翻转，十有八九位置是错的。

### 5. 横屏项目别忘了测左右

竖屏项目最容易先注意顶部刘海，横屏项目则更容易在左右两侧出问题。
所以横屏一定要重点看 `conformX` 的结果。

### 6. Editor 里看起来正常，不代表真机就正常

`Screen.safeArea` 在编辑器里通常没有小游戏平台那套真实坐标数据。
尤其是微信、抖音这类 SDK 场景，最终一定要上真机或平台预览里确认。

## 总结

这套实现的关键只有一句话：

```text
先拿到正确的安全区域，再把它转换成 RectTransform 的锚点
```
