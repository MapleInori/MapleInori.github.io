# 2D 随机方向 / 位置 / 力度 完整总结

在 Unity 2D 开发中，经常需要生成随机方向、随机位置或随机力度的效果。
本文总结三种常见需求，并给出「手动计算」和「Unity 封装方法」两种实现方式。

## 三种需求场景

1. **纯随机方向**（长度固定 1，用于移动、反弹、朝向）
2. **圆内随机位置**（用于生成点、出生点、爆炸范围）
3. **随机方向 + 随机力度**（散射、击退、爆炸推力）

---

# 1. 纯随机方向（长度 = 1）

> 适用场景：需要知道一个均匀分布的随机方向，但不需要关心距离。

## 方法 A：手动计算角度（最原始）

```csharp
// 随机一个 0~360 度的角度
float angle = Random.Range(0f, 360f);

// 将角度转换为弧度（Mathf 的三角函数使用弧度制）
float rad = angle * Mathf.Deg2Rad;

// 通过极坐标公式将弧度转换为单位向量
// Mathf.Cos(rad) 对应 x 分量，Mathf.Sin(rad) 对应 y 分量
// 结果向量长度恒为 1，相当于单位圆上的一个随机点
Vector2 dir = new Vector2(Mathf.Cos(rad), Mathf.Sin(rad));
```

## 方法 B：Unity 封装方法（推荐）

```csharp
// Random.insideUnitCircle 返回一个圆内的随机点（长度 0~1）
// .normalized 将其归一化，得到长度为 1 的随机方向向量
Vector2 dir = Random.insideUnitCircle.normalized;
```

**效果完全一样**：均匀 360° 方向，向量长度 = 1。

> ✅ 推荐使用方法 B，代码更简洁，可读性更好。

---

# 2. 圆内随机位置（长度 0~radius）

> 适用场景：在指定半径范围内生成随机点，如怪物出生点、随机掉落位置等。

## 方法 A：手动计算角度 + 随机半径

```csharp
// 随机一个 0~360 度的角度，并直接转换为弧度
float angle = Random.Range(0f, 360f) * Mathf.Deg2Rad;

// 随机一个半径值（0~5 之间），决定点在圆内的距离中心多远
float radius = Random.Range(0f, 5f);

// 极坐标转直角坐标系：方向向量 × 半径
// 结果是一个距离原点 0~5 的随机位置
Vector2 pos = new Vector2(Mathf.Cos(angle), Mathf.Sin(angle)) * radius;
```

## 方法 B：Unity 封装方法（最简单）

```csharp
// Random.insideUnitCircle 直接返回单位圆内的一个随机点
// 乘以 5f 将其缩放到半径为 5 的圆内
Vector2 pos = Random.insideUnitCircle * 5f;
```

**效果一样**：圆内均匀分布，不需要手动计算角度。

> ✅ 推荐使用方法 B，避免手动处理弧度和极坐标转换。

---

# 3. 随机方向 + 随机力度（方向均匀、大小随机）

> 适用场景：爆炸击退、箭塔散射、随机推力等需要「方向随机 + 大小随机」的效果。

## 方法 A：手动计算角度

```csharp
// 随机一个角度并转为弧度
float angle = Random.Range(0f, 360f) * Mathf.Deg2Rad;

// 将角度转为单位方向向量
Vector2 dir = new Vector2(Mathf.Cos(angle), Mathf.Sin(angle));

// 随机一个力度值（3~8 之间）
float force = Random.Range(3f, 8f);

// 方向 × 力度 = 最终的力向量
Vector2 forceVec = dir * force;
```

## 方法 B：Unity 封装方法

```csharp
// 获取一个随机的单位方向向量
Vector2 dir = Random.insideUnitCircle.normalized;

// 随机一个力度值
float force = Random.Range(3f, 8f);

// 方向 × 力度 = 最终的力向量
Vector2 forceVec = dir * force;
```

**效果一样**：方向均匀分布，力度在 min~max 之间随机。

---

# 极简一句话总结

* **纯方向**

  * 手动：角度 → Cos/Sin → 得到单位向量
  * Unity：`Random.insideUnitCircle.normalized`
* **圆内位置**

  * 手动：角度 + 随机半径 → 极坐标转直角坐标
  * Unity：`Random.insideUnitCircle * radius`
* **方向 + 力度**

  * 手动：角度转向量 × 随机力
  * Unity：`Random.insideUnitCircle.normalized × 随机力`

---

# 最实用的最终口诀

* **要方向**：必须 `.normalized`（归一化到长度 1）
* **要位置**：直接乘半径（不需要归一化）
* **要力度**：方向 × 随机大小
* **永远不要**直接用 `new Vector2(Random.Range(-1f, 1f), Random.Range(-1f, 1f))` 当方向用

> ⚠️ 原因：这种方式会导致方向分布不均匀（角落区域概率更高），而圆分布的随机方向才是真正均匀的。
