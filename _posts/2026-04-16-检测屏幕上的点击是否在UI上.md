# 检测屏幕上的点击是否在 UI 上

这篇主要记录两种常见情况：

1. 判断某个指定的 UI 元素有没有被点到。
2. 判断当前这次点击是不是点在任意 UI 上。

这两个需求看起来很像，但其实适合的方法不完全一样。

## 方法一：判断某个具体 UI 是否被点到

如果我已经知道要判断的是哪一个 UI，例如某个按钮面板、某个弹窗区域，最直接的办法就是用 `RectTransformUtility.RectangleContainsScreenPoint`。

```csharp
public static bool RectangleContainsScreenPoint(
    RectTransform rect,
    Vector2 screenPoint,
    Camera cam
);
```

它的意思很直接：给它一个 `RectTransform`，再给它一个屏幕坐标点，它会告诉我这个点是否落在这个 UI 矩形范围内。

### 参数

| 参数 | 含义 |
| --- | --- |
| `rect` | 要测试的 `RectTransform`。 |
| `screenPoint` | 要测试的屏幕坐标点。 |
| `cam` | 用来做这次判断的摄像机。`Screen Space - Overlay` 模式下通常可以传 `null`。 |

### 返回值

如果这个屏幕点在目标 UI 的矩形范围内，返回 `true`；否则返回 `false`。

### 适合什么时候用

- 已经明确知道要判断哪一个 UI。
- 只关心这个指定区域有没有被点击。
- 想做“点击弹窗外区域关闭弹窗”这一类逻辑。

### 示例

```csharp
public RectTransform panel;

public bool IsClickOnPanel()
{
    return RectTransformUtility.RectangleContainsScreenPoint(
        panel,
        Input.mousePosition,
        null
    );
}
```

这个方法的优点是简单直接，但它只能判断“这一个 RectTransform 的范围”，并不能告诉我屏幕上是不是点到了别的 UI。

## 方法二：判断当前点击是否落在任意 UI 上

如果需求不是“有没有点到某个指定 UI”，而是“这次点击是不是点到了任何 UI”，那我更常用的是走 `EventSystem` 的 UI 射线检测。

这也是我在其他项目里更常用的一种写法。

```csharp
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.EventSystems;

public bool IsPointerOverUI()
{
    if (EventSystem.current == null)
        return false;

    PointerEventData eventData = new PointerEventData(EventSystem.current)
    {
        position = Input.mousePosition
    };

    List<RaycastResult> results = new List<RaycastResult>();
    EventSystem.current.RaycastAll(eventData, results);

    return results.Count > 0;
}
```

这段代码的思路是：

1. 先构造一个当前指针位置的 `PointerEventData`。
2. 再让 `EventSystem` 用这个位置去做一次 UI 射线检测。
3. 只要结果列表里有命中的对象，就说明这次点击落在某个 UI 上。

## 这两种方法的区别

`RectangleContainsScreenPoint` 更像是：

- “我已经知道要检查哪个 UI 了，请告诉我点有没有落在它上面。”

`EventSystem.RaycastAll` 更像是：

- “我不关心具体是哪一个 UI，只想知道这次点击有没有打到 UI。”

所以这两个方法不是谁替代谁，而是适合不同场景。

## 实际使用时怎么选

- 如果要判断“是否点击到某个面板 / 某个按钮区域”，优先用 `RectangleContainsScreenPoint`。
- 如果要判断“点击 UI 时不要触发场景交互”，更适合用 `EventSystem.RaycastAll` 或类似的 UI 命中检测方法。

比如角色移动、场景点击、拖拽物体时，通常都会先判断一次“当前点击是不是在 UI 上”。如果是，就先拦住后面的世界点击逻辑。

## 几个容易忽略的小点

- 这种基于 `EventSystem` 的写法，前提是场景里要有 `EventSystem`。
- UI 要能被射线检测到，相关图形组件本身也要允许被 Raycast。
- 如果项目是触屏而不是鼠标输入，就不能直接只写 `Input.mousePosition`，而要改成对应触摸点的位置。
- `RectangleContainsScreenPoint` 只判断矩形区域，不会自动替你处理“是否被别的 UI 挡住”这种问题。

## 这次的理解

现在回头看，这个问题其实可以拆成一句更清楚的话：

不是单纯地问“有没有点到 UI”，而是先问自己“我要判断的是某一个指定 UI，还是任意 UI”。

一旦把这个问题分清楚，方法选择就会自然很多：

- 指定 UI：`RectTransformUtility.RectangleContainsScreenPoint`
- 任意 UI：`EventSystem.RaycastAll`
