## 什么是序列化

简单来说，序列化就是把 Unity 中的一些数据，例如场景、预制体、脚本数据，转换成可以存储或者传输的形式，然后写入磁盘文件或者通过网络发送出去。

反序列化则是把这些已经保存好的数据重新读回来，还原成程序可用的对象。

Unity 中的场景文件、预制体、`ScriptableObject` 文件，本质上都是以 YAML 形式存储的。

如果只从 JSON 角度理解，也可以把它记成：

- 序列化：把类对象转换成字符串
- 反序列化：把字符串转换回类对象

## Unity 中的序列化字段

- `public` 字段默认可以被序列化。
- `private` 字段必须加 `[SerializeField]` 才能被序列化。
- 没有初始化的私有字段，通常不会被序列化到文件里；公有字段即使没手动初始化，也可能以默认值写入文件。
- 这里说的初始化，就是给变量赋一个初始值；如果不赋值，就会使用系统默认值。
- 要让自定义 C# 类能被 Unity 序列化，给类加上 `[Serializable]` 就行。

## Unity 展示数据的流程

Unity 展示 Inspector 数据时，大致可以这样理解：

1. 先读取脚本本身，判断哪些字段允许被序列化，并拿到这些字段的默认值。
2. 再去看对应的序列化文件里，这些字段有没有实际保存下来的值。
3. 如果有，就用文件中的值覆盖脚本里的默认值。

## 结构体序列化

结构体本身是支持序列化的，只要里面的字段也符合 Unity 的序列化规则就可以。

## 关于 MonoBehaviour 在 Inspector 里编辑

一个 `MonoBehaviour` 默认只能在它自己的 Inspector 面板里编辑。

当然，也可以通过 Editor 脚本，强行在一个 Mono 的 Inspector 里去编辑另一个 Mono 的数据，但通常不建议这样做，因为结构会变得比较绕。

## 多态序列化

多态，也就是“父类引用指向子类实例”，默认情况下不能直接被 Unity 序列化，因为多态是运行时特性。

Unity 2019.3 之后可以用 `[SerializeReference]` 来处理这类情况，接口类型也是类似思路。不过这类写法一般也要谨慎使用。

## 为什么不序列化字典

Unity 默认不序列化字典，主要原因之一是字典的顺序不稳定。

即使元素内容没变，顺序也可能变化；一旦顺序变化，版本控制里就会显示差异，很容易引发合并冲突。

## Newtonsoft.Json

`Newtonsoft.Json` 功能更全，支持字典、格式化输出、多态等，但需要额外导入。

### JsonConvert.SerializeObject

序列化：对象 -> JSON 字符串

它本质上是基于 .NET 反射和自定义解析规则来工作的，支持几乎所有常见的 C# 类型，比如字典、多态、枚举、`null` 值等，也能自定义序列化规则，例如忽略字段、格式化输出、处理循环引用。

```csharp
public static string SerializeObject(object value, Formatting formatting);
```

| 参数           | 含义                                                                                                               |
| -------------- | ------------------------------------------------------------------------------------------------------------------ |
| `value`      | 要序列化的对象。支持大多数 C# 类型，例如字典、枚举、多态对象、`null` 值等，不要求对象必须加 `[Serializable]`。 |
| `formatting` | 格式化选项。`Formatting.None` 表示单行压缩，`Formatting.Indented` 表示换行缩进。                               |

### JsonConvert.DeserializeObject

反序列化：JSON 字符串 -> 对象

它会先解析 JSON 字符串，再按目标类型去匹配字段并创建对象。处理复杂类型时，也可以依靠额外规则做兼容转换。

```csharp
public static T DeserializeObject<T>(string value); // 泛型版
public static object DeserializeObject(string value, Type type); // 非泛型版
```

| 参数       | 含义                                                           |
| ---------- | -------------------------------------------------------------- |
| `value`  | 要解析的 JSON 字符串，支持包含字典、`null`、多态等复杂内容。 |
| `T/type` | 目标类型。支持大多数 C# 类型，例如字典、自定义类、枚举等。     |

### 序列化与反序列化示例

```csharp
using UnityEngine;
using Newtonsoft.Json;

public class NewtonSoftDemo : MonoBehaviour
{
    void Start()
    {
        PlayerData player = new PlayerData();
        player.playerName = "小红";
        player.level = 20;
        player.hp = 199.0f;

        string jsonStr = JsonConvert.SerializeObject(player, Formatting.Indented);
        Debug.Log("NewtonSoft序列化后的JSON：\n" + jsonStr);
        // {
        //   "playerName": "小红",
        //   "level": 20,
        //   "hp": 199.0
        // }

        PlayerData newPlayer = JsonConvert.DeserializeObject<PlayerData>(jsonStr);
        Debug.Log("NewtonSoft反序列化后的血量：" + newPlayer.hp); // 199.0
    }
}
```

## UnityEngine.JsonUtility

`JsonUtility` 是 Unity 自带的 JSON 工具，比较轻量，但只支持带 `[Serializable]` 的类或结构体，不支持字典、多态等更复杂的情况。

### JsonUtility.ToJson

序列化：对象 -> JSON 字符串

可以把它理解成：遍历对象字段，然后把字段名和值按 JSON 规则拼接成字符串。它更适合处理基础类型和结构明确的简单数据类。

```csharp
public static string ToJson(object obj, bool prettyPrint = false);
```

| 参数            | 含义                                                                                                                     |
| --------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `obj`         | 要序列化的对象。通常要求是带 `[Serializable]` 的类或结构体实例，`MonoBehaviour` 和 `GameObject` 不能直接这样处理。 |
| `prettyPrint` | 可选参数，默认是 `false`。`true` 时输出格式化 JSON，`false` 时输出压缩的单行 JSON。                                |

### JsonUtility.FromJson

反序列化：JSON 字符串 -> 对象

它会根据字段名去匹配目标类中的字段，再把 JSON 中的数据填回对象里。字段名不匹配或者类型不兼容时，通常不会直接报错，而是保留默认值。

```csharp
public static T FromJson<T>(string json); // 泛型版
public static object FromJson(string json, Type type); // 非泛型版
```

| 参数       | 含义                                                               |
| ---------- | ------------------------------------------------------------------ |
| `json`   | 要解析的 JSON 字符串。格式不合法时，结果可能为空或者只保留默认值。 |
| `T/type` | 目标类型。通常要求是带 `[Serializable]` 的类或结构体。           |

### 序列化与反序列化示例

```csharp
using UnityEngine;

[System.Serializable]
public class PlayerData
{
    public string playerName;
    public int level;
    public float hp;
}

public class JsonUtilityDemo : MonoBehaviour
{
    void Start()
    {
        PlayerData player = new PlayerData();
        player.playerName = "小明";
        player.level = 10;
        player.hp = 99.5f;

        string jsonStr = JsonUtility.ToJson(player);
        Debug.Log("序列化后的JSON：" + jsonStr);
        // {"playerName":"小明","level":10,"hp":99.5}

        PlayerData newPlayer = JsonUtility.FromJson<PlayerData>(jsonStr);
        Debug.Log("反序列化后的玩家名：" + newPlayer.playerName); // 小明
        Debug.Log("反序列化后的等级：" + newPlayer.level); // 10
    }
}
```
