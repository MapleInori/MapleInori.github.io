## 1. Lua 简介

Lua 是一种轻量级脚本语言，适合嵌入其他应用程序。游戏开发中，可以使用 C# 或 C++ 实现底层和性能敏感的部分，再使用 Lua 编写需要频繁调整的业务逻辑。

Unity 项目接入 xLua 等框架后，可以让 Lua 调用 Unity 和 C# 提供的能力。这样，一部分逻辑可以不跟随完整安装包一起更新。

### 编译型语言与解释型语言

编译型语言通常先将源代码编译为可执行文件，再运行编译结果，例如 C、C++、C#。

Lua 属于脚本语言。运行 Lua 文件时，需要由 Lua 解释器读取并执行代码。它适合快速开发和调整，但性能敏感的底层逻辑通常仍由宿主语言负责。

## 2. 环境搭建

安装 Lua 解释器后，将解释器所在目录加入 Windows 的 `Path` 环境变量。重新打开命令行，执行：

```bash
lua -v
```

如果能够输出 Lua 版本，说明配置已经生效。

新建 `01.HelloWorld.lua`：

```lua
print("HelloWorld!")
```

在脚本所在目录执行：

```bash
lua 01.HelloWorld.lua
```

运行结果：

```text
HelloWorld!
```

## 3. 基础语法

### 3.1 注释

新建 `02.comments.lua`：

```lua
-- 这是单行注释

--[[
    这是多行注释
    可以写多行内容
]]
```

Lua 的单行注释以 `--` 开头，多行注释写在 `--[[` 和 `]]` 之间，也可以写成 `--[[`和 `--]]`.

### 3.2 变量与作用域

Lua 会根据赋值自动判断变量类型，不需要像 C# 一样提前声明类型。

新建 `03.variable.lua`：

```lua
-- 未使用 local 修饰时，默认定义全局变量
name = "Lua"
print(name)

-- 未定义的变量，其值为 nil
print(id)

-- 将变量赋值为 nil，可以理解为移除这个变量保存的值
name = nil
print(name)

-- 使用 local 定义局部变量
local version = "5.3"
print(version)

-- Lua 内置变量：查看当前 Lua 版本
print(_VERSION)
```

一般情况下，应优先使用 `local`。全局变量可以被其他代码修改，项目变大后不容易追踪。

`local` 变量的作用域从声明位置开始，到当前代码块结束。代码块可以是当前文件，也可以是函数、循环或条件分支。

### 3.3 数据类型

使用 `type()` 可以获取一个值的类型。`type()` 的返回值是字符串。

新建 `04.data_type.lua`：

```lua
local name = "Unity"

print(type(name))        -- string
print(type(type(name)))  -- string
print(type(123))         -- number
print(type(1.0))         -- number
print(type(true))        -- boolean
print(type(nil))         -- nil
print(type(function() end)) -- function
print(type({}))          -- table
```

常用的数据类型：

| 类型         | 说明                               |
| ------------ | ---------------------------------- |
| `nil`      | 表示没有值，类似 C# 中的 `null`  |
| `boolean`  | 布尔值，包括 `true` 和 `false` |
| `number`   | 数字类型                           |
| `string`   | 字符串                             |
| `function` | 函数                               |
| `table`    | 表，可以表达数组、字典和对象       |

### 3.4 字符串

Lua 支持单引号、双引号和多行字符串。字符串拼接使用 `..`。

新建 `05.string.lua`：

```lua
local str1 = "abc"
local str2 = 'def'
local str3 = [[
one
two
]]

print(str1 .. str2)              -- abcdef
print(#str1)                     -- 3
print(string.upper(str1))        -- ABC
print(string.lower("LUA"))       -- lua
print(string.reverse("abc"))     -- cba
print(string.sub("abcdef", 3))   -- cdef
print(string.sub("abcdef", 3, 5)) -- cde
print(string.format("player %d", 1)) -- player 1
print(string.rep("abc", 2))      -- abcabc
print(string.char(65))           -- A
print(string.byte("A"))          -- 65
```

Lua 字符串的索引从 `1` 开始。`string.find()` 可以返回多个值：

```lua
local startIndex, endIndex = string.find("abcdefg", "cde")
print(startIndex, endIndex) -- 3  5
```

`string.gsub()` 也会返回多个值，分别是替换后的字符串和替换次数：

```lua
local result, count = string.gsub("abcd", "bc", "**")
print(result, count) -- a**d  1
```

虽然 Lua 可以将部分数字字符串自动转换为数字，但业务代码中不建议依赖隐式转换。需要转换时，可以使用 `tonumber()` 和 `tostring()`。

```lua
print(tonumber("5") + tonumber("2")) -- 7
print("player_" .. tostring(1))      -- player_1
```

### 3.5 运算符

新建 `07.operator.lua`：

```lua
print(2 ^ 3)  -- 8，乘方
print(2 ~= 3) -- true，不等于

local num = 1
num = num + 1
print(num)    -- 2
```

与 C# 不同，Lua 没有 `++`、`--`、`+=` 和 `-=`。例如自增操作需要写成 `num = num + 1`。

## 4. Table

`table` 是 Lua 中非常重要的数据结构。它既可以用作数组，也可以使用字符串或其他值作为键，表达类似字典或对象的数据。

### 4.1 数组与索引

新建 `06.array.lua`：

```lua
local data = {
    "abc",
    123,
    [-1] = 100,
    [0] = 99,
    [4] = 233
}

print(data[1])  -- abc
print(data[2])  -- 123
print(data[-1]) -- 100
print(data[0])  -- 99
print(data[4])  -- 233

data[1] = "def"
print(data[1])  -- def

local matrix = {
    { "aa", "bb" },
    { 11, 22 }
}

print(matrix[2][1]) -- 11
```

Lua 数组通常从索引 `1` 开始，但 `table` 实际上可以使用其他数字作为键，也允许索引不连续。

只有当数字索引从 `1` 开始连续排列时，才适合使用 `#data` 获取长度。对于存在空洞或额外键的 `table`，不要依赖 `#data` 判断元素总数。

### 4.2 使用 ipairs 和 pairs 遍历 Table

Lua 的 `table` 可以同时保存数组形式的数据和键值对形式的数据。遍历时，需要根据希望读取的内容选择 `ipairs()` 或 `pairs()`。

新建 `12.iterator.lua`：

```lua
-- one = "cc" 等价于 ["one"] = "cc"
-- 没有显式填写键的 "aa" 和 "bb"，会自动使用索引 1 和 2
-- [4] 和 [-1] 是显式指定的数字键
-- ["two"] 是显式指定的字符串键
local data = {
    one = "cc",
    "aa",
    "bb",
    [4] = 3,
    [-1] = 4,
    ["two"] = "dd"
}

for key, value in ipairs(data) do
    print("key:" .. key .. ", value:" .. value)
end
```

创建完成后，`data` 中实际保存了这些键值对：

| 键        | 值       | 说明                              |
| --------- | -------- | --------------------------------- |
| `1`     | `"aa"` | 未指定键，自动使用数字索引        |
| `2`     | `"bb"` | 未指定键，自动使用数字索引        |
| `4`     | `3`    | 显式指定数字键，索引 `3` 不存在 |
| `-1`    | `4`    | 显式指定负数键                    |
| `"one"` | `"cc"` | 省略中括号的字符串键              |
| `"two"` | `"dd"` | 显式书写的字符串键                |

下面两种字符串键写法效果相同：

```lua
local data1 = {
    one = "cc"
}

local data2 = {
    ["one"] = "cc"
}

print(data1.one)      -- cc
print(data2["one"])   -- cc
```

只有当字符串符合 Lua 标识符规则时，才能使用 `one = "cc"` 和 `data.one` 这种简写。如果字符串中包含空格等特殊字符，需要使用中括号：

```lua
local data = {
    ["player name"] = "Maple"
}

print(data["player name"])
```

#### 使用 ipairs 遍历连续数组

`ipairs(data)` 从数字索引 `1` 开始，按照 `1`、`2`、`3` 的顺序依次读取。当遇到第一个值为 `nil` 的索引时，遍历结束。

```lua
for key, value in ipairs(data) do
    print("key:" .. key .. ", value:" .. value)
end
```

运行结果：

```text
key:1, value:aa
key:2, value:bb
```

因为索引 `3` 不存在，读取到 `data[3]` 时得到 `nil`，遍历结束。即使 `data[4]` 存在，`ipairs()` 也不会继续读取。

`ipairs()` 也不会读取 `-1`、`"one"` 和 `"two"` 这些键。它适合遍历从索引 `1` 开始连续排列的数组。

可以把下面的泛型 `for` 循环理解成类似 C# 的 `foreach`：

```lua
for key, value in ipairs(data) do
    -- key 是当前索引
    -- value 是当前索引对应的值
end
```

不过，`ipairs()` 本身不是“指向 table 的指针”。更准确地说，它会返回泛型 `for` 循环需要的迭代信息，让循环按顺序取得键和值。

#### 使用 pairs 遍历所有键值对

`pairs(data)` 可以遍历 `table` 中的所有键值对，包括不连续数字键、负数键和字符串键：

```lua
for key, value in pairs(data) do
    print("key:" .. key .. ", value:" .. value)
end
```

输出结果可能类似：

```text
key:1, value:aa
key:2, value:bb
key:one, value:cc
key:two, value:dd
key:4, value:3
key:-1, value:4
```

需要注意，`pairs()` **不保证遍历顺序**。上面的输出只是其中一种可能结果，不应该依赖它处理有顺序要求的逻辑。

#### 统计不规则 Table 中的键值对

如果 `table` 包含不连续索引或字符串键，不应使用 `#data` 获取元素总数。可以使用 `pairs()` 遍历并主动计数：

```lua
local count = 0

for _ in pairs(data) do
    count = count + 1
end

print(count) -- 6
```

这里得到的是 `table` 中的键值对总数，不是数组长度。

简单记：

| 写法             | 适用场景                            | 是否保证顺序                     |
| ---------------- | ----------------------------------- | -------------------------------- |
| `ipairs(data)` | 遍历从 `1` 开始连续排列的数组部分 | 按数字索引递增                   |
| `pairs(data)`  | 遍历 `table` 中的全部键值对       | 不保证顺序                       |
| `#data`        | 获取连续数组部分的长度              | 不适合存在空洞的不规则 `table` |

### 4.3 Table 成员与成员函数

新建 `14.table.lua`：

```lua
local player = {
    name = "Maple",
    level = 1
}

print(player["name"])
print(player.name)

player.sayHello = function()
    print("Hello")
end

player.sayHello()
```

当字符串键符合变量命名规则时，`player.name` 是 `player["name"]` 的简写。

成员函数可以使用 `self` 访问当前 `table`。下面两种定义方式效果相同：

```lua
local player = {
    name = "Maple"
}

player.sayName = function(self) --显示传入self，需要用冒号调用
    print(self.name)
end

player:sayName()

function player:sayHello()
    print("Hello, " .. self.name)
end

player:sayHello()
```

冒号调用 `player:sayName()` 相当于 `player.sayName(player)`，Lua 会自动把当前 `table` 作为第一个参数传入。

## 5. 流程控制

### 5.1 if 条件判断

新建 `08.if.lua`：

```lua
local level = 10

if level >= 20 then
    print("高级")
elseif level >= 10 then
    print("中级")
else
    print("初级")
end
```

Lua 使用 `then` 开始条件分支，使用 `end` 结束整个结构。Lua 没有 `switch`，需要时可以使用 `if ... elseif ... else`。

### 5.2 while 循环

新建 `09.while.lua`：

```lua
local num = 1

while num < 3 do
    print(num)
    num = num + 1
end
```

`while` 会先判断条件，再决定是否执行循环体。

### 5.3 repeat until 循环

新建 `10.repeat.lua`：

```lua
local num = 1

repeat
    print(num)
    num = num + 1
until num > 3
```

`repeat until` 会先执行一次循环体，再判断退出条件。条件为 `true` 时结束循环。

Lua 支持使用 `break` 跳出循环，但没有内置的 `continue`。

### 5.4 for 循环

新建 `11.for.lua`：

```lua
local data = { "aa", "bb", "cc" }

for index = 1, #data do
    print(data[index])
end

for index = #data, 1, -1 do
    print(data[index])
end
```

数字 `for` 循环的格式是：

```lua
for 变量 = 初始值, 结束值, 步长 do
    -- 循环体
end
```

步长可以省略，默认值为 `1`。

## 6. 函数

Lua 中的函数是一种值，可以赋给变量，也可以保存到 `table` 中。

新建 `13.function.lua`：

```lua
function sayHello()
    print("Hello")
end

sayHello()

local add = function(a, b)
    return a + b
end

print(add(5, 7)) -- 12
```

Lua 按照代码顺序执行。调用一个函数前，需要确保它已经完成定义。

### 6.1 参数数量

Lua 调用函数时，实参数量不必严格等于形参数量：

```lua
local function show(a, b)
    print(a, b)
end

show(1)       -- 1  nil
show(1, 2, 3) -- 1  2
```

缺少的参数会得到 `nil`，多出的普通参数不会被使用。

### 6.2 可变参数

使用 `...` 可以接收数量不固定的参数：

```lua
local function sum(...)
    local args = { ... }
    local total = 0

    for _, value in ipairs(args) do
        total = total + value
    end

    return total
end

print(sum(1, 2, 3))       -- 6
print(sum(1, 2, 3, 4, 5)) -- 15
```

### 6.3 多返回值

Lua 函数可以同时返回多个值：

```lua
local function getPosition()
    return 10, 20
end

local x, y = getPosition()
print(x, y) -- 10  20
```

## 7. 多脚本加载：require

随着代码增加，需要将不同功能拆分到多个文件中。Lua 可以使用 `require()` 加载模块。

`require()` 不只是简单地执行另一个脚本。它还会负责：

```text
根据模块名查找文件
-> 执行第一次加载到的模块
-> 保存模块返回值
-> 缓存加载结果
-> 后续加载同名模块时复用缓存
```

目录结构：

```text
project/
├── 15.require.lua
└── subfile/
    ├── config.lua
    └── subfile1.lua
```

### 7.1 查看模块搜索路径

`15.require.lua` 首先打印 `package.path`：

```lua
print(package.path)
```

`package.path` 是 Lua 查找普通 Lua 模块时使用的搜索规则。输出结果通常包含多个路径模板，例如：

```text
.\?.lua;.\?\init.lua
```

其中的 `?` 会被替换为传给 `require()` 的模块名。

例如：

```lua
require("subfile/subfile1")
```

Lua 会根据 `package.path` 中的模板尝试查找类似下面的文件：

```text
.\subfile\subfile1.lua
```

调用 `require()` 时通常不需要填写 `.lua` 扩展名，因为搜索模板中已经包含了 `.lua`。

需要注意，搜索路径通常相对于程序启动时的**当前工作目录**解析，不一定相对于正在执行的 `.lua` 文件。如果从不同目录启动程序，模块查找结果也可能不同。

### 7.2 加载并执行子文件

`subfile/subfile1.lua`：

```lua
print("subfile1")
```

`15.require.lua` 中第一次加载这个文件：

```lua
require("subfile/subfile1")
```

Lua 找到 `subfile/subfile1.lua` 后，会执行文件中的代码，因此控制台输出：

```text
subfile1
```

模块名更常见的写法是使用点号分隔层级：

```lua
require("subfile.subfile1")
```

在常见配置下，它也会查找：

```text
subfile/subfile1.lua
```

点号不是不能使用的特殊字符。对于模块层级，推荐优先使用 `subfile.subfile1` 这种写法。

### 7.3 使用 package.loaded 缓存模块

成功加载模块后，Lua 会将加载结果缓存在 `package.loaded` 中。键是传给 `require()` 的模块名：

```lua
require("subfile/subfile1")

print(package.loaded["subfile/subfile1"])
```

`subfile1.lua` 没有主动返回值，因此 `require()` 会使用 `true` 表示模块已经成功加载。输出结果类似：

```text
subfile1
true
```

再次使用相同模块名加载时：

```lua
require("subfile/subfile1")
require("subfile/subfile1")
```

第二次调用会直接返回 `package.loaded["subfile/subfile1"]` 中的缓存，不会再次执行 `print("subfile1")`。

如果主动清除缓存：

```lua
package.loaded["subfile/subfile1"] = nil
require("subfile/subfile1")
```

Lua 会重新查找并执行模块，因此再次输出：

```text
subfile1
```

日常开发中，不建议随意清除模块缓存。模块初始化逻辑重复执行后，可能产生难以排查的问题。

### 7.4 模块名不同，缓存键也不同

课程示例中还使用了：

```lua
require("./subfile/subfile1")
```

这种写法不推荐使用。`require()` 接收的是**模块名**，不是普通文件路径。Lua 查找模块时，会将模块名中的点号转换为目录分隔符。因此，开头的 `.` 不应直接理解为文件路径中的“当前目录”。

如果确实使用不同写法加载模块，还要注意缓存键不同：

```lua
package.loaded["subfile/subfile1"]
package.loaded["./subfile/subfile1"]
```

对 `require()` 来说，这是两个不同的模块名。如果自定义搜索规则让它们最终指向同一个文件，它们仍然可能分别执行一次并产生两份缓存。

为了避免同一个文件被重复加载，项目中应该统一模块名写法。推荐写成：

```lua
require("subfile.subfile1")
```

不要在同一个项目中混用：

```lua
require("subfile.subfile1")
require("subfile/subfile1")
require("./subfile/subfile1")
```

### 7.5 使用 return 导出模块内容

如果子文件只有：

```lua
local appName = "hxsd"
```

其他脚本不能直接读取这个局部变量。模块可以创建一个 `table`，将需要公开的数据放入表中，再通过 `return` 返回。

`subfile/config.lua`：

```lua
local config = {}

config.appName = "hxsd"

return config
```

在 `15.require.lua` 中接收模块返回值：

```lua
local config = require("subfile.config")

print(config.appName) -- hxsd
```

执行流程：

```text
require("subfile.config")
-> 根据 package.path 找到 subfile/config.lua
-> 执行 config.lua
-> config.lua 返回 config 表
-> require() 将 config 表写入 package.loaded["subfile.config"]
-> 主文件使用局部变量 config 接收这个表
-> 通过 config.appName 读取公开数据
```

这种写法很适合用来组织配置和功能模块。模块内部数据优先使用 `local`，只通过返回的 `table` 公开外部真正需要访问的内容，可以减少全局变量被意外覆盖的问题。

### 7.6 完整示例

结合三个脚本，可以将 `15.require.lua` 整理为：

```lua
-- 查看 Lua 模块搜索路径
print(package.path)

print("-----------------------")

-- 第一次加载：找到并执行 subfile/subfile1.lua
require("subfile.subfile1")

-- subfile1.lua 没有 return，因此缓存值为 true
print(package.loaded["subfile.subfile1"])

-- 再次加载同名模块：直接使用缓存，不会重复执行
require("subfile.subfile1")

-- 加载配置模块，并接收它返回的 table
local config = require("subfile.config")

print(config.appName) -- hxsd
```

运行结果类似：

```text
.\?.lua;.\?\init.lua;...
-----------------------
subfile1
true
hxsd
```

简单记：

| 内容                            | 作用                             |
| ------------------------------- | -------------------------------- |
| `require("subfile.config")`   | 查找、执行并缓存模块             |
| `package.path`                | 普通 Lua 模块的搜索规则          |
| `package.loaded`              | 已加载模块的缓存表               |
| `return config`               | 将模块希望公开的数据返回给调用者 |
| `local config = require(...)` | 接收模块导出的内容               |

## 8. 元表：metatable

Lua 的 `table` 可以保存数据和函数。除此之外，还可以为一个 `table` 绑定另一个 `table`，用来定义额外行为。这个用于扩展行为的表叫作**元表**（metatable）。

元表本身仍然是普通的 `table`。它的特殊之处在于，可以定义以双下划线开头的**元方法**（metamethod）。当 Lua 遇到对应操作时，会自动调用这些元方法。

例如：

| 元方法         | 触发场景                               |
| -------------- | -------------------------------------- |
| `__tostring` | 将值转换为字符串，例如调用 `print()` |
| `__add`      | 使用 `+` 运算符                      |
| `__index`    | 读取表中不存在的键                     |
| `__newindex` | 为表中不存在的键赋值                   |

这一节先使用 `__tostring`，让 `table` 打印出便于阅读的内容。

### 8.1 直接打印 Table

新建 `16.metatable.lua`：

```lua
local t1 = { 1, 2, 3 }
local t2 = { 2, 3, 4 }

-- 直接打印 table 时，通常只能看到类型和地址信息。
print(t1)
```

运行结果类似：

```text
table: 000001F...
```

`t1` 中明明保存了 `1`、`2` 和 `3`，但直接打印时看不到这些元素。为了方便调试，希望输出结果更接近：

```text
{1, 2, 3, }
```

### 8.2 定义 __tostring 元方法

创建元表 `meta`，并在其中定义 `__tostring`：

```lua
local meta = {
    -- 当绑定该元表的 table 需要转换为字符串时，Lua 会调用这个函数。
    -- 参数 target 是当前准备转换为字符串的 table。
    __tostring = function(target)
        local format = "{"

        -- 遍历 table 中的键值对，将每个值拼接到字符串中。
        for _, value in pairs(target) do
            format = format .. value .. ", "
        end

        format = format .. "}"
        return format
    end
}
```

这里的 `meta` 仍然是普通 `table`。真正重要的是其中的 `__tostring` 函数。

函数执行过程：

```text
创建 format = "{"
-> 使用 pairs() 遍历 target
-> 将每个 value 和 ", " 追加到 format
-> 在末尾追加 "}"
-> 返回最终字符串
```

当 `target` 是 `{ 1, 2, 3 }` 时，拼接过程类似：

| 步骤         | `format` 的值   |
| ------------ | ----------------- |
| 初始值       | `"{"`           |
| 拼接 `1`   | `"{1, "`        |
| 拼接 `2`   | `"{1, 2, "`     |
| 拼接 `3`   | `"{1, 2, 3, "`  |
| 拼接右大括号 | `"{1, 2, 3, }"` |

当前写法会在最后一个元素后面留下 `", "`。这不影响理解，后续可以再优化格式。

### 8.3 使用 setmetatable 绑定元表

定义元表后，使用 `setmetatable()` 将它绑定到目标 `table`：

```lua
-- 参数 1：需要扩展行为的 table。
-- 参数 2：提供扩展规则的元表。
setmetatable(t1, meta)
```

绑定后，`t1` 中保存的数据没有变化：

```lua
print(t1[1]) -- 1
print(t1[2]) -- 2
print(t1[3]) -- 3
```

变化的是：当 Lua 需要把 `t1` 转换为字符串时，会先检查元表中是否定义了 `__tostring`。

### 8.4 多个 Table 可以共享同一个元表

课程脚本将 `meta` 同时绑定到 `t1` 和 `t2`：

```lua
setmetatable(t1, meta)
setmetatable(t2, meta)
```

这样两个表都会使用相同的字符串转换规则：

```lua
print(t1)
print(t2)
```

运行结果类似：

```text
{1, 2, 3, }
{2, 3, 4, }
```

这里可以把 `meta` 理解成一套可复用的规则。每个目标 `table` 仍然保存自己的数据，但可以共享元表中定义的行为。

如果需要查看一个表当前绑定的元表，可以使用：

```lua
print(getmetatable(t1) == meta) -- true
print(getmetatable(t2) == meta) -- true
```

### 8.5 print(t1) 的完整调用流程

绑定元表后，再次执行：

```lua
print(t1)
```

完整流程：

```text
print(t1)
↓
print() 需要将 t1 转换为字符串
↓
发现 t1 绑定了元表 meta
↓
找到 meta.__tostring
↓
调用 meta.__tostring(t1)
↓
参数 target 接收到 t1
↓
pairs(target) 遍历 t1 中的数据
↓
返回 "{1, 2, 3, }"
↓
print() 输出字符串
```

`__tostring` 中形参的名字可以自定义。课程脚本中使用 `tt`，这里改成了更容易理解的 `target`。无论叫什么，它接收到的都是当前需要转换为字符串的表。

### 8.6 完整示例

结合课程脚本，可以整理为：

```lua
local t1 = { 1, 2, 3 }
local t2 = { 2, 3, 4 }

-- 绑定元表前，直接打印 table 通常只能看到地址信息。
print(t1)

local meta = {
    -- 当 table 需要转换为字符串时，自动调用 __tostring。
    __tostring = function(target)
        local format = "{"

        for _, value in pairs(target) do
            format = format .. value .. ", "
        end

        format = format .. "}"
        return format
    end
}

-- t1 和 t2 共享同一套元表规则。
setmetatable(t1, meta)
setmetatable(t2, meta)

print(t1)
print(t2)
```

运行结果类似：

```text
table: 000001F...
{1, 2, 3, }
{2, 3, 4, }
```

### 8.7 当前示例的限制

课程示例适合帮助理解元表，但还不是通用的 `table` 格式化工具。

需要注意：

- `pairs()` 不保证遍历顺序。示例输出通常符合数组顺序，但业务逻辑不应该依赖这一点。
- 示例只拼接 `value`，不会显示键。
- 如果值本身是嵌套 `table`，直接执行字符串拼接可能报错。
- 如果表中包含复杂值，需要判断类型并递归处理。
- 示例会在最后一个元素后面留下额外的逗号和空格。

对于元素连续排列的简单数组，可以使用 `table.concat()` 写出更简洁的版本：

```lua
local meta = {
    __tostring = function(target)
        return "{" .. table.concat(target, ", ") .. "}"
    end
}
```

运行结果：

```text
{1, 2, 3}
```

不过，`table.concat()` 也主要适用于连续数组，无法完整展示字符串键、不连续索引或嵌套 `table`。

### 8.8 延伸：使用 __add 自定义加法

元表不仅能修改字符串转换行为，还可以通过 `__add` 自定义 `+` 运算符。

例如，后面的练习题会让两个数组合并：

```lua
local result = data1 + data2
```

当 `data1` 绑定的元表中存在 `__add` 时，Lua 会将这段代码转换为类似下面的调用：

```lua
local result = meta.__add(data1, data2)
```

理解 `__tostring` 的调用流程后，再看 `__add` 会比较自然：都是在元表中定义规则，再由 Lua 在特定操作发生时自动调用。

## 9. 练习题

1. 编写函数，统计不规则 `table` 中键值对的总数。
2. 编写一个接收可变参数的函数，使用冒泡排序法排序，并返回参数个数和排序结果。
3. 使用元表的 `__add` 元方法，实现两个数组内容的合并。

### 9.1 统计不规则 Table 的键值对总数

不规则 `table` 可能包含字符串键、不连续的数字键或负数键，因此不能使用 `#data` 统计全部元素。可以使用 `pairs()` 遍历并主动计数：

```lua
local function countEntries(data)
    local count = 0

    for _ in pairs(data) do
        count = count + 1
    end

    return count
end

local data = {
    "aa",
    "bb",
    [4] = "dd",
    [-1] = "negative",
    name = "Lua"
}

print(countEntries(data)) -- 5
```

这里统计的是键值对总数，而不是数组长度。

### 9.2 使用冒泡排序处理可变参数

使用 `{ ... }` 可以将可变参数保存到数组中，再通过冒泡排序交换相邻元素：

```lua
local function bubbleSort(...)
    local data = { ... }
    local count = #data

    for i = 1, count - 1 do
        for j = 1, count - i do
            if data[j] > data[j + 1] then
                data[j], data[j + 1] = data[j + 1], data[j]
            end
        end
    end

    return count, data
end

local count, result = bubbleSort(5, 3, 8, 1, 2)

print(count)                  -- 5
print(table.concat(result, ", ")) -- 1, 2, 3, 5, 8
```

`data[j], data[j + 1] = data[j + 1], data[j]` 利用了 Lua 的多变量赋值，不需要额外声明临时变量。

### 9.3 使用 __add 合并两个数组

通过元表的 `__add` 元方法，可以自定义两个数组执行 `+` 运算时的行为：

```lua
local arrayMeta = {
    __add = function(left, right)
        local result = {}
-- 如果是连续数据，可以用ipairs，拼接后的表是按顺序的，
-- 如果不是连续数据，则要用pairs，不过拼起来的顺序无法保证
        for _, value in ipairs(left) do
            table.insert(result, value)
        end

        for _, value in ipairs(right) do
            table.insert(result, value)
        end

        return setmetatable(result, arrayMeta)
    end
}

local data1 = setmetatable({ 1, 2, 3 }, arrayMeta)
local data2 = setmetatable({ 4, 5, 6 }, arrayMeta)

local result = data1 + data2

print(table.concat(result, ", ")) -- 1, 2, 3, 4, 5, 6
```

执行 `data1 + data2` 时，Lua 会调用 `arrayMeta.__add(data1, data2)`。示例创建并返回一个新数组，因此不会修改原来的 `data1` 和 `data2`。
