## 1. Lua 简介

Lua 是一种轻量级脚本语言，适合嵌入其他应用程序。游戏开发中，可以使用 C# 或 C++ 实现底层和性能敏感的部分，再使用 Lua 编写需要频繁调整的业务逻辑。

Unity 项目接入 xLua 等框架后，可以让 Lua 调用 Unity 和 C# 提供的能力。这样，一部分逻辑可以不跟随完整安装包一起更新。

### 编译型语言与解释型语言

编译型语言通常先将源代码编译为可执行文件，再运行编译结果，例如 C、C++、C#。

Lua 属于脚本语言。运行 Lua 文件时，需要由 Lua 解释器读取并执行代码。它适合快速开发和调整，但性能敏感的底层逻辑通常仍由宿主语言负责。

## 2. 环境搭建

安装 Lua 解释器后，将解释器所在目录加入 Windows 的 `Path` 环境变量。重新打开命令行，执行：

```bash
lua -v
```

如果能够输出 Lua 版本，说明配置已经生效。

新建 `01.HelloWorld.lua`：

```lua
print("HelloWorld!")
```

在脚本所在目录执行：

```bash
lua 01.HelloWorld.lua
```

运行结果：

```text
HelloWorld!
```

## 3. 基础语法

### 3.1 注释

新建 `02.comments.lua`：

```lua
-- 这是单行注释

--[[
    这是多行注释
    可以写多行内容
]]
```

Lua 的单行注释以 `--` 开头，多行注释写在 `--[[` 和 `]]` 之间，也可以写成 `--[[`和 `--]]`.

### 3.2 变量与作用域

Lua 会根据赋值自动判断变量类型，不需要像 C# 一样提前声明类型。

新建 `03.variable.lua`：

```lua
-- 未使用 local 修饰时，默认定义全局变量
name = "Lua"
print(name)

-- 未定义的变量，其值为 nil
print(id)

-- 将变量赋值为 nil，可以理解为移除这个变量保存的值
name = nil
print(name)

-- 使用 local 定义局部变量
local version = "5.3"
print(version)

-- Lua 内置变量：查看当前 Lua 版本
print(_VERSION)
```

一般情况下，应优先使用 `local`。全局变量可以被其他代码修改，项目变大后不容易追踪。

`local` 变量的作用域从声明位置开始，到当前代码块结束。代码块可以是当前文件，也可以是函数、循环或条件分支。

### 3.3 数据类型

使用 `type()` 可以获取一个值的类型。`type()` 的返回值是字符串。

新建 `04.data_type.lua`：

```lua
local name = "Unity"

print(type(name))        -- string
print(type(type(name)))  -- string
print(type(123))         -- number
print(type(1.0))         -- number
print(type(true))        -- boolean
print(type(nil))         -- nil
print(type(function() end)) -- function
print(type({}))          -- table
```

常用的数据类型：

| 类型         | 说明                               |
| ------------ | ---------------------------------- |
| `nil`      | 表示没有值，类似 C# 中的 `null`  |
| `boolean`  | 布尔值，包括 `true` 和 `false` |
| `number`   | 数字类型                           |
| `string`   | 字符串                             |
| `function` | 函数                               |
| `table`    | 表，可以表达数组、字典和对象       |

### 3.4 字符串

Lua 支持单引号、双引号和多行字符串。字符串拼接使用 `..`。

新建 `05.string.lua`：

```lua
local str1 = "abc"
local str2 = 'def'
local str3 = [[
one
two
]]

print(str1 .. str2)              -- abcdef
print(#str1)                     -- 3
print(string.upper(str1))        -- ABC
print(string.lower("LUA"))       -- lua
print(string.reverse("abc"))     -- cba
print(string.sub("abcdef", 3))   -- cdef
print(string.sub("abcdef", 3, 5)) -- cde
print(string.format("player %d", 1)) -- player 1
print(string.rep("abc", 2))      -- abcabc
print(string.char(65))           -- A
print(string.byte("A"))          -- 65
```

Lua 字符串的索引从 `1` 开始。`string.find()` 可以返回多个值：

```lua
local startIndex, endIndex = string.find("abcdefg", "cde")
print(startIndex, endIndex) -- 3  5
```

`string.gsub()` 也会返回多个值，分别是替换后的字符串和替换次数：

```lua
local result, count = string.gsub("abcd", "bc", "**")
print(result, count) -- a**d  1
```

虽然 Lua 可以将部分数字字符串自动转换为数字，但业务代码中不建议依赖隐式转换。需要转换时，可以使用 `tonumber()` 和 `tostring()`。

```lua
print(tonumber("5") + tonumber("2")) -- 7
print("player_" .. tostring(1))      -- player_1
```

### 3.5 运算符

新建 `07.operator.lua`：

```lua
print(2 ^ 3)  -- 8，乘方
print(2 ~= 3) -- true，不等于

local num = 1
num = num + 1
print(num)    -- 2
```

与 C# 不同，Lua 没有 `++`、`--`、`+=` 和 `-=`。例如自增操作需要写成 `num = num + 1`。

## 4. Table

`table` 是 Lua 中非常重要的数据结构。它既可以用作数组，也可以使用字符串或其他值作为键，表达类似字典或对象的数据。

### 4.1 数组与索引

新建 `06.array.lua`：

```lua
local data = {
    "abc",
    123,
    [-1] = 100,
    [0] = 99,
    [4] = 233
}

print(data[1])  -- abc
print(data[2])  -- 123
print(data[-1]) -- 100
print(data[0])  -- 99
print(data[4])  -- 233

data[1] = "def"
print(data[1])  -- def

local matrix = {
    { "aa", "bb" },
    { 11, 22 }
}

print(matrix[2][1]) -- 11
```

Lua 数组通常从索引 `1` 开始，但 `table` 实际上可以使用其他数字作为键，也允许索引不连续。

只有当数字索引从 `1` 开始连续排列时，才适合使用 `#data` 获取长度。对于存在空洞或额外键的 `table`，不要依赖 `#data` 判断元素总数。

### 4.2 使用 ipairs 和 pairs 遍历 Table

`ipairs` 从索引 `1` 开始依次读取。当中间遇到缺失的索引时，遍历会停止。

新建 `12.iterator.lua`：

```lua
local data = {
    "aa",
    "bb",
    [4] = "dd",
    name = "Lua"
}

for index, value in ipairs(data) do
    print(index, value)
end
```

运行结果：

```text
1    aa
2    bb
```

因为索引 `3` 不存在，所以不会继续读取索引 `4`。

`pairs` 可以遍历所有键值对：

```lua
for key, value in pairs(data) do
    print(key, value)
end
```

使用 `pairs` 时，不应依赖输出顺序。如果需要统计一个不规则 `table` 中的所有键值对，可以主动计数：

```lua
local count = 0

for _ in pairs(data) do
    count = count + 1
end

print(count) -- 4
```

### 4.3 Table 成员与成员函数

新建 `14.table.lua`：

```lua
local player = {
    name = "Maple",
    level = 1
}

print(player["name"])
print(player.name)

player.sayHello = function()
    print("Hello")
end

player.sayHello()
```

当字符串键符合变量命名规则时，`player.name` 是 `player["name"]` 的简写。

成员函数可以使用 `self` 访问当前 `table`。下面两种定义方式效果相同：

```lua
local player = {
    name = "Maple"
}

player.sayName = function(self)
    print(self.name)
end

player:sayName()

function player:sayHello()
    print("Hello, " .. self.name)
end

player:sayHello()
```

冒号调用 `player:sayName()` 相当于 `player.sayName(player)`，Lua 会自动把当前 `table` 作为第一个参数传入。

## 5. 流程控制

### 5.1 if 条件判断

新建 `08.if.lua`：

```lua
local level = 10

if level >= 20 then
    print("高级")
elseif level >= 10 then
    print("中级")
else
    print("初级")
end
```

Lua 使用 `then` 开始条件分支，使用 `end` 结束整个结构。Lua 没有 `switch`，需要时可以使用 `if ... elseif ... else`。

### 5.2 while 循环

新建 `09.while.lua`：

```lua
local num = 1

while num < 3 do
    print(num)
    num = num + 1
end
```

`while` 会先判断条件，再决定是否执行循环体。

### 5.3 repeat until 循环

新建 `10.repeat.lua`：

```lua
local num = 1

repeat
    print(num)
    num = num + 1
until num > 3
```

`repeat until` 会先执行一次循环体，再判断退出条件。条件为 `true` 时结束循环。

Lua 支持使用 `break` 跳出循环，但没有内置的 `continue`。

### 5.4 for 循环

新建 `11.for.lua`：

```lua
local data = { "aa", "bb", "cc" }

for index = 1, #data do
    print(data[index])
end

for index = #data, 1, -1 do
    print(data[index])
end
```

数字 `for` 循环的格式是：

```lua
for 变量 = 初始值, 结束值, 步长 do
    -- 循环体
end
```

步长可以省略，默认值为 `1`。

## 6. 函数

Lua 中的函数是一种值，可以赋给变量，也可以保存到 `table` 中。

新建 `13.function.lua`：

```lua
function sayHello()
    print("Hello")
end

sayHello()

local add = function(a, b)
    return a + b
end

print(add(5, 7)) -- 12
```

Lua 按照代码顺序执行。调用一个函数前，需要确保它已经完成定义。

### 6.1 参数数量

Lua 调用函数时，实参数量不必严格等于形参数量：

```lua
local function show(a, b)
    print(a, b)
end

show(1)       -- 1  nil
show(1, 2, 3) -- 1  2
```

缺少的参数会得到 `nil`，多出的普通参数不会被使用。

### 6.2 可变参数

使用 `...` 可以接收数量不固定的参数：

```lua
local function sum(...)
    local args = { ... }
    local total = 0

    for _, value in ipairs(args) do
        total = total + value
    end

    return total
end

print(sum(1, 2, 3))       -- 6
print(sum(1, 2, 3, 4, 5)) -- 15
```

### 6.3 多返回值

Lua 函数可以同时返回多个值：

```lua
local function getPosition()
    return 10, 20
end

local x, y = getPosition()
print(x, y) -- 10  20
```

## 7. 多脚本加载：require

随着代码增加，需要将功能拆分到多个文件中。Lua 可以使用 `require()` 加载模块。

目录结构：

```text
project/
├── 15.require.lua
└── subfile/
    └── config.lua
```

`subfile/config.lua`：

```lua
local config = {
    appName = "Lua Demo",
    version = "1.0.0"
}

return config
```

`15.require.lua`：

```lua
print(package.path)

local config = require("subfile.config")

print(config.appName)
print(config.version)
```

调用 `require("subfile.config")` 时，不需要填写 `.lua` 扩展名。Lua 会根据 `package.path` 中的规则查找模块。

成功加载模块后，Lua 会将结果缓存在 `package.loaded` 中。再次加载同一个模块时，通常会直接复用缓存：

```lua
print(package.loaded["subfile.config"])
```

如果确实需要重新执行模块代码，可以清除对应缓存：

```lua
package.loaded["subfile.config"] = nil
local config = require("subfile.config")
```

日常开发中，不建议随意清除模块缓存。模块初始化逻辑重复执行后，可能产生难以排查的问题。

## 8. 元表：metatable

直接打印一个 `table` 时，Lua 通常只会显示它的地址。通过元表中的 `__tostring` 元方法，可以定义它转换为字符串时的行为。

新建 `16.metatable.lua`：

```lua
local data = { 1, 2, 3 }

local meta = {
    __tostring = function(target)
        return "{" .. table.concat(target, ", ") .. "}"
    end
}

setmetatable(data, meta)

print(data)
```

运行结果：

```text
{1, 2, 3}
```

调用 `print(data)` 时，Lua 会自动调用 `__tostring`。元表还可以定义其他行为，例如使用 `__add` 自定义两个 `table` 相加时的逻辑。

### 8.1 执行流程

首先，下面的代码创建了一个普通的数组形式 `table`：

```lua
local data = { 1, 2, 3 }
```

它相当于：

```lua
data[1] = 1
data[2] = 2
data[3] = 3
```

如果此时直接调用 `print(data)`，通常只能看到类似 `table: 000001F...` 的地址信息。

接着创建元表 `meta`，并在其中定义 `__tostring` 元方法：

```lua
local meta = {
    __tostring = function(target)
        return "{" .. table.concat(target, ", ") .. "}"
    end
}
```

当 Lua 需要将一个对象转换为字符串时，如果对象的元表中存在 `__tostring`，Lua 就会调用这个函数。参数 `target` 是准备转换为字符串的对象。

然后，为 `data` 绑定元表：

```lua
setmetatable(data, meta)
```

绑定后，`data` 中保存的数据没有变化，但它可以使用 `meta` 中定义的扩展规则。

最后执行：

```lua
print(data)
```

`print()` 需要将 `data` 转换为字符串，因此会调用：

```lua
meta.__tostring(data)
```

`table.concat(target, ", ")` 使用 `", "` 连接数组中的元素，得到 `"1, 2, 3"`。再在首尾拼接大括号，最终返回 `"{1, 2, 3}"`。

完整调用链：

```text
print(data)
↓
发现 data 绑定了元表 meta
↓
找到 meta.__tostring
↓
调用 meta.__tostring(data)
↓
table.concat(data, ", ") 得到 "1, 2, 3"
↓
返回 "{1, 2, 3}"
↓
print 输出结果
```

这个示例适用于元素连续排列的数组。如果 `table` 包含字符串键、嵌套表或不连续索引，`table.concat()` 无法完整展示全部内容。

## 9. 练习题

1. 编写函数，统计不规则 `table` 中键值对的总数。
2. 编写一个接收可变参数的函数，使用冒泡排序法排序，并返回参数个数和排序结果。
3. 使用元表的 `__add` 元方法，实现两个数组内容的合并。

### 9.1 统计不规则 Table 的键值对总数

不规则 `table` 可能包含字符串键、不连续的数字键或负数键，因此不能使用 `#data` 统计全部元素。可以使用 `pairs()` 遍历并主动计数：

```lua
local function countEntries(data)
    local count = 0

    for _ in pairs(data) do
        count = count + 1
    end

    return count
end

local data = {
    "aa",
    "bb",
    [4] = "dd",
    [-1] = "negative",
    name = "Lua"
}

print(countEntries(data)) -- 5
```

这里统计的是键值对总数，而不是数组长度。

### 9.2 使用冒泡排序处理可变参数

使用 `{ ... }` 可以将可变参数保存到数组中，再通过冒泡排序交换相邻元素：

```lua
local function bubbleSort(...)
    local data = { ... }
    local count = #data

    for i = 1, count - 1 do
        for j = 1, count - i do
            if data[j] > data[j + 1] then
                data[j], data[j + 1] = data[j + 1], data[j]
            end
        end
    end

    return count, data
end

local count, result = bubbleSort(5, 3, 8, 1, 2)

print(count)                  -- 5
print(table.concat(result, ", ")) -- 1, 2, 3, 5, 8
```

`data[j], data[j + 1] = data[j + 1], data[j]` 利用了 Lua 的多变量赋值，不需要额外声明临时变量。

### 9.3 使用 __add 合并两个数组

通过元表的 `__add` 元方法，可以自定义两个数组执行 `+` 运算时的行为：

```lua
local arrayMeta = {
    __add = function(left, right)
        local result = {}

        for _, value in ipairs(left) do
            table.insert(result, value)
        end

        for _, value in ipairs(right) do
            table.insert(result, value)
        end

        return setmetatable(result, arrayMeta)
    end
}

local data1 = setmetatable({ 1, 2, 3 }, arrayMeta)
local data2 = setmetatable({ 4, 5, 6 }, arrayMeta)

local result = data1 + data2

print(table.concat(result, ", ")) -- 1, 2, 3, 4, 5, 6
```

执行 `data1 + data2` 时，Lua 会调用 `arrayMeta.__add(data1, data2)`。示例创建并返回一个新数组，因此不会修改原来的 `data1` 和 `data2`。
