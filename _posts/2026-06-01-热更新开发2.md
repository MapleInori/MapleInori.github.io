## 1. Lua 简介

Lua 是一种轻量级脚本语言，适合嵌入其他应用程序。游戏开发中，可以使用 C# 或 C++ 实现底层和性能敏感的部分，再使用 Lua 编写需要频繁调整的业务逻辑。

Unity 项目接入 xLua 等框架后，可以让 Lua 调用 Unity 和 C# 提供的能力。这样，一部分逻辑可以不跟随完整安装包一起更新。

### 编译型语言与解释型语言

编译型语言通常先将源代码编译为可执行文件，再运行编译结果，例如 C、C++、C#。

Lua 属于脚本语言。运行 Lua 文件时，需要由 Lua 解释器读取并执行代码。它适合快速开发和调整，但性能敏感的底层逻辑通常仍由宿主语言负责。

## 2. 环境搭建

安装 Lua 解释器后，将解释器所在目录加入 Windows 的 `Path` 环境变量。重新打开命令行，执行：

HelloWorld!

## 3. 基础语法

### 3.1 注释

新建 `02.comments.lua`。下面同时列出 C# 的注释写法作为对比：

```lua
-- // C#单行注释
-- Lua的单行注释（两个减号）

--[[
    /*
        这是C#的多行注释
    */
]]

--[[
    这是Lua的多行注释
]]

--[[
    这也是Lua的多行注释
--]]     -- 注意：前面多加了一个减号会导致多行注释的结束语法不被识别
```

> **C# vs Lua 注释对比**：
>
> |      | C#             | Lua                                    |
> | ---- | -------------- | -------------------------------------- |
> | 单行 | `// 注释`    | `-- 注释`                            |
> | 多行 | `/* 注释 */` | `--[[ 注释 ]]` 或 `--[[ 注释 --]]` |
>
> 注意 `--[[]]` 中如果写成 `---[[` 多了一个减号，就不会被识别为多行注释。

### 3.2 变量与作用域

Lua 会根据赋值自动判断变量类型，不需要像 C# 一样提前声明类型。

新建 `03.variable.lua`。教程里保留了 C# 变量声明的对比，方便理解 Lua 的区别：

```lua
-- 变量声明
-- C#定义变量
-- 类型 变量名 = 变量值
-- string name = "hxsd";

-- Lua会自动推导变量类型，不需要写类型
-- Lua可以不用写分号，也可以写分号
name = "hxsd"

-- 调用变量
-- C# Debug.Log()  Console.WriteLine()
print(name)

-- 内置变量
-- 特点：下划线加大写字母
-- 查看Lua的版本
print(_VERSION)

-- 可以调用一个不存在的变量
-- 值是nil，等同于C#中的null
print(id)

-- 销毁一个已经定义的变量：赋值为 nil
name = nil
print(name)

-- 全局变量和局部变量
-- data = "hxsd" 定义了一个全局变量
-- 一般开发中，轻易不会使用全局变量，
-- 因为全局变量可以被随意修改，不安全，不稳定
local data = "hxsd" -- 定义了一个局部变量（data的作用域是当前文件）
print(data)
```

一般情况下，应优先使用 `local`。全局变量可以被其他代码修改，项目变大后不容易追踪。

`local` 变量的作用域从声明位置开始，到当前代码块结束。代码块可以是当前文件，也可以是函数、循环或条件分支。

### 3.3 数据类型

使用 `type()` 可以获取一个值的类型。`type()` 的返回值是字符串。

新建 `04.data_type.lua`。教程里额外演示了"对未定义变量取类型"和"判断未定义变量"的写法：

```lua
local name = "Unity"

-- 获得变量的类型，使用type()
-- type()是函数结构，是一个代码片段，可以重复调用执行，有参数有返回值
print(type(name))

-- 获得type()返回值的类型
-- type()返回值的类型是string
print(type(type(name)))

-- 对一个没有定义的变量获取类型——返回 "nil" 字符串，而不是报错
print(type(bb))

-- 判断一个变量是否未定义
print(type(bb) == "nil")  -- true

-- 数字类型的数值
-- C#中多种数字类型（int, float, double...），在Lua中被归纳到number数据类型下
print(type(123))
print(type(1.0))
-- 布尔类型的数值
-- 布尔有true和false
print(type(true))
-- 单引号字符串也是 string
print(type('hxsd'))

```

常用的数据类型：

| 类型         | 说明                               |
| ------------ | ---------------------------------- |
| `nil`      | 表示没有值，类似 C# 中的 `null`  |
| `boolean`  | 布尔值，包括 `true` 和 `false` |
| `number`   | 数字类型                           |
| `string`   | 字符串                             |
| `function` | 函数                               |
| `table`    | 表，可以表达数组、字典和对象       |

### 3.4 字符串

Lua 支持单引号、双引号和多行字符串。字符串拼接使用 `..`。

新建 `05.string.lua`。教程里的示例更全，保留所有原始注释：

```lua
-- 单行定义字符串
local str1 = "abc"
local str2 = 'def'

-- 多行定义字符串
-- 完全的还原字符串内部的定义结构，如换行符都是存储的
local str3 = [[
one
two
]]

print(str3)

-- C#字符串拼接用 +，Lua用 ..
-- str1 + str2
print(str1 .. str2)

-- C#写多了要注意！
-- Lua对两个数字格式的字符串做 "+" 时，会将它们转换为数字类型再相加
-- 转换失败会报错，转换成功则得到数字结果
print("5" + "2")  -- 输出 7（数字），不是 "52"！

-- 获取字符串的长度
print(#str1)

------------------------------------------------------------------------

-- 字符串相关函数
-- string.函数名()

-- 字符串变大写
print(string.upper(str1))
-- 字符串变小写
print(string.lower("LUA"))

-- Lua支持多返回值
-- Lua字符串的下标是从1开始的！！！

-- 字符串查找
-- 参数1：被查找的字符串
-- 参数2：查找的内容
-- 返回值：起始找到的位置下标 和 结束找到的位置下标
local startIndex, endIndex = string.find("abcdefg", "cde")
print(startIndex, endIndex)  -- 3  5

-- 字符串反转
print(string.reverse("abcdefg"))

-- 截取字符串
-- 方式1（提供起始位置下标，截取到结尾）
print(string.sub("abcdefg", 3))     -- cdefg

-- 方式2（提供起始 + 结束下标）注意：C#的第二个参数是截取长度，Lua是结束下标
print(string.sub("abcdefg", 3, 6))  -- cdef

-- 截取到倒数第二个字符
local data = "abcdefghijklmn"
print(string.sub(data, 3, #data - 1))

-- 字符串格式化
-- 参数1：需要被格式化的字符串，其中包含占位符，%d表示数字
-- 参数2：填入格式化字符串中的内容
print(string.format("Im the %d player, other is %d", 1, 2))

-- 字符串重复
-- 参数1：被重复的字符串
-- 参数2：重复的次数
print(string.rep("abc", 2))  -- abcabc

-- ASCII码转字符
print(string.char(65))        -- A
-- 字符转ASCII码
print(string.byte("A"))       -- 65

-- 字符串替换
-- 参数1：原始串
-- 参数2：需要替换的内容
-- 参数3：替换后的内容
-- 返回值：替换结果, 替换的次数
local result, count = string.gsub("abcd", "bc", "**")
print(result, count)          -- a**d  1

Lua 字符串的索引从 `1` 开始。`string.find()` 可以返回多个值：

local startIndex, endIndex = string.find("abcdefg", "cde")
print(startIndex, endIndex) -- 3  5
```

`string.gsub()` 也会返回多个值，分别是替换后的字符串和替换次数：

```lua
local result, count = string.gsub("abcd", "bc", "**")
print(result, count) -- a**d  1
```

虽然 Lua 可以将部分数字字符串自动转换为数字，但业务代码中不建议依赖隐式转换。需要转换时，可以使用 `tonumber()` 和 `tostring()`。

```lua
print(tonumber("5") + tonumber("2")) -- 7
print("player_" .. tostring(1))      -- player_1
```

## 4. Table

`table` 是 Lua 中非常重要的数据结构。它既可以用作数组，也可以使用字符串或其他值作为键，表达类似字典或对象的数据。

### 4.1 数组与索引

新建 `06.array.lua`：

```lua
-- 数组初始化
-- C# string[] data = new string[20];

local data = {}

-- 起始索引是1开始
-- 类型可以混合
-- 索引值可以为负数
-- 即使索引从1开始，也可以赋值0索引
-- 索引可以断开
-- 初始化时，对于没有指定索引的值，索引是从1向上累加的
-- 初始化提供索引的赋值方法：[索引值] = 数值
data = {"abc", 123, [-1] = 100, [0] = 99, [4] = 233}

print(data[1])   -- abc
print(data[2])   -- 123
print(data[-1])  -- 100
print(data[0])   -- 99
print(data[4])   -- 233

-- 获取数组的长度
-- #data 获得的是从1索引开始、索引连续的数据个数，中间断开则计数结束
-- 这种方式不稳定，对于有空洞的table不要依赖它
print(#data)     -- 2（只数到1和2，索引3不存在就停了）

-- 修改某一个值
data[1] = "def"
print(data[1])   -- def

-- 多维数组
local data2 = { {"aa", "bb"}, {11, 22} }
print(data2[2][1]) -- 11

-- 数组在Lua中是用Table实现的
```

### 4.2 使用 ipairs 和 pairs 遍历 Table

Lua 的 `table` 可以同时保存数组形式的数据和键值对形式的数据。遍历时，需要根据希望读取的内容选择 `ipairs()` 或 `pairs()`。

新建 `12.iterator.lua`。教程代码保留了完整的注释和预期输出：

```lua
-- 迭代器（遍历table）
-- one是不加中括号的字符串索引
-- "aa" "bb" 自动加1，2索引
-- [4]指定数字索引
-- [-1]指定负数索引
-- ["two"]是加中括号的字符串索引写法
local data = {one = "cc", "aa", "bb", [4] = 3, [-1] = 4, ["two"] = "dd"}

-- 连续索引数值迭代器
-- 此处的for结构，可以理解为C#的foreach结构
-- 连续数字索引迭代器，只会获取从1开始的数字索引，且必须索引连续才能持续获得值
for k, v in ipairs(data)
do
    print("k:" .. k .. ", v:" .. v)
end

print("-----------------------------")

-- 所有数值迭代器（pairs）
-- 相对于ipairs，去掉了"i"——i可以理解为int，去掉int的迭代器也就去掉了连续数值索引的限定
-- 获取table长度的最稳定方法，就是使用pairs遍历计数
for k, v in pairs(data)
do
    print("K:" .. k .. ", V:" .. v)
end

--[[  预期输出：
k:1, v:aa
k:2, v:bb
-----------------------------
K:1, V:aa
K:2, V:bb
K:one, V:cc
K:two, V:dd
K:4, V:3
K:-1, V:4
--]]

```

创建完成后，`data` 中实际保存了这些键值对：

| 键        | 值       | 说明                              |
| --------- | -------- | --------------------------------- |
| `1`     | `"aa"` | 未指定键，自动使用数字索引        |
| `2`     | `"bb"` | 未指定键，自动使用数字索引        |
| `4`     | `3`    | 显式指定数字键，索引 `3` 不存在 |
| `-1`    | `4`    | 显式指定负数键                    |
| `"one"` | `"cc"` | 省略中括号的字符串键              |
| `"two"` | `"dd"` | 显式书写的字符串键                |

下面两种字符串键写法效果相同：

```lua
local data1 = {
    one = "cc"
}

local data2 = {
    ["one"] = "cc"
}

print(data1.one)      -- cc
print(data2["one"])   -- cc
```

只有当字符串符合 Lua 标识符规则时，才能使用 `one = "cc"` 和 `data.one` 这种简写。如果字符串中包含空格等特殊字符，需要使用中括号：

```lua
local data = {
    ["player name"] = "Maple"
}

print(data["player name"])
```

#### 使用 ipairs 遍历连续数组

`ipairs(data)` 从数字索引 `1` 开始，按照 `1`、`2`、`3` 的顺序依次读取。当遇到第一个值为 `nil` 的索引时，遍历结束。

```lua
for key, value in ipairs(data) do
    print("key:" .. key .. ", value:" .. value)
end
```

运行结果：

```text
key:1, value:aa
key:2, value:bb
```

因为索引 `3` 不存在，读取到 `data[3]` 时得到 `nil`，遍历结束。即使 `data[4]` 存在，`ipairs()` 也不会继续读取。

`ipairs()` 也不会读取 `-1`、`"one"` 和 `"two"` 这些键。它适合遍历从索引 `1` 开始连续排列的数组。

可以把下面的泛型 `for` 循环理解成类似 C# 的 `foreach`：

```lua
for key, value in ipairs(data) do
    -- key 是当前索引
    -- value 是当前索引对应的值
end
```

不过，`ipairs()` 本身不是“指向 table 的指针”。更准确地说，它会返回泛型 `for` 循环需要的迭代信息，让循环按顺序取得键和值。

#### 使用 pairs 遍历所有键值对

`pairs(data)` 可以遍历 `table` 中的所有键值对，包括不连续数字键、负数键和字符串键：

```lua
for key, value in pairs(data) do
    print("key:" .. key .. ", value:" .. value)
end
```

输出结果可能类似：

```text
key:1, value:aa
key:2, value:bb
key:one, value:cc
key:two, value:dd
key:4, value:3
key:-1, value:4
```

需要注意，`pairs()` **不保证遍历顺序**。上面的输出只是其中一种可能结果，不应该依赖它处理有顺序要求的逻辑。

#### 统计不规则 Table 中的键值对

如果 `table` 包含不连续索引或字符串键，不应使用 `#data` 获取元素总数。可以使用 `pairs()` 遍历并主动计数：

```lua
local count = 0

for _ in pairs(data) do
    count = count + 1
end

print(count) -- 6
```

这里得到的是 `table` 中的键值对总数，不是数组长度。

简单记：

| 写法             | 适用场景                            | 是否保证顺序                     |
| ---------------- | ----------------------------------- | -------------------------------- |
| `ipairs(data)` | 遍历从 `1` 开始连续排列的数组部分 | 按数字索引递增                   |
| `pairs(data)`  | 遍历 `table` 中的全部键值对       | 不保证顺序                       |
| `#data`        | 获取连续数组部分的长度              | 不适合存在空洞的不规则 `table` |

### 4.3 Table 成员与成员函数

新建 `14.table.lua`。教程里演示了两种 `self` 调用写法——显式传参和隐式传参：

```lua
-- Table支持数字索引存储数据
-- Table支持字符串索引（关联索引）存储数据

local data = {one = "cc", "aa", "bb", [4] = 3, [-1] = 4, ["two"] = "dd"}

print(data[2])        -- bb
print(data["one"])    -- cc
print(data.two)       -- dd（符合命名规则的字符串键可以用 . 访问）

-- 因为函数是一种数据类型，所以可以把函数存在table的键下
data.func1 = function()
    print("data表中的func1函数")
end

data.func1()          -- 调用表中存储的函数

--------------------------------------------------------------

data.func2 = function()
    print("func2:" .. data.two)
end

data.func2()

--------------------------------------------------------------

-- 第一种self调用写法
-- 成员函数定义时，显式加入self变量，对应C#的this关键字
-- 函数内部可以通过self变量获取当前table的其他值或函数
data.func3 = function(self)
    print("func3:" .. self.two)
end

-- 调用时，必须使用 ":"，因为 ":" 调用会对self关键字赋值
data:func3()

--------------------------------------------------------------

-- 第二种self调用写法
-- 隐式给self赋值（定义时用 : 代替 .）
function data:func4()
    print("func4:" .. self.two)
end

data:func4()
```

### 5.1 if 条件判断

新建 `08.if.lua`。教程里每个 Lua 分支结构旁边都标了 C# 等价写法：

```lua
--[[ C# 单if
    if(条件)
    {
        条件达成的语句块
    }
]]

local con1 = true

if(con1)
then
    print("条件1达成")
end

--[[ C# if-else
    if(条件)
    {
        条件达成的语句块
    }
    else
    {
        否则达成的语句块
    }
]]

if(false)
then
    print("条件1达成")
else
    print("else达成")
end

--[[ C# if-elseif-else
    if(条件)
    {
        条件达成的语句块
    }

    {
        else if条件
    }
    else
    {
        否则达成的语句块
    }

if(false)

    print("条件1达成")
elseif(true)

    print("elseif达成")

    print("else达成")
end

    if(条件)
    {

        {
            //进入第二层条件
        }
    }

    if(true)
    then
        print("进入第二层if")
    end
end

-- Lua没有switch，所以可以使用if-elseif-else的结构代替
```

### 5.2 while 循环

新建 `09.while.lua`：

```lua
--[[ C# while
    while(条件)
    {
        循环体
    }
]]

local num = 1

while(num < 3) -- 当条件满足时，进入循环
do
    -- local修饰的num，虽然是局部变量，但是作用域为当前文件，
    -- 所以在循环体内，可以取得值

```

`while` 会先判断条件，再决定是否执行循环体。

### 5.3 repeat until 循环

新建 `10.repeat.lua`。教程里还把 `repeat until` 当成 C# 的 `do-while` 来对比，并演示了 `break` 跳出循环：

```lua
--[[ C# do while
    do
    {
        循环体
    }
    while(条件)

    至少执行一次do结构体内的代码，while条件满足时，再do执行代码
]]

local num = 1

repeat
    print(num)
    num = num + 1
until(num > 5) -- 直到条件满足时，跳出循环

-------------------------------------------------------------

-- break会跳出循环，没有continue
-- 下面的结构模拟了 C# 中 do { if break; if break; } while(false) 的"只执行一次+中途跳出"用法
repeat
    if(true)
    then
        print("此处跳出")
        break
    end

    if(true)
    then
        print("此2处跳出")
        break
    end
until(true)

print("继续执行")  -- break 后从这里继续
```

### 5.4 for 循环

新建 `11.for.lua`：

```lua
local data = {"aa", "bb", "cc", "dd", "ee"}

-- 参数1：变量i的初始值，遍历Lua表，使用1
-- 参数2：增长到多少
-- [参数3]：增长步长，默认值是1
for i = 1, #data
do
    print(data[i])
end

-- 倒序遍历
for i = #data, 1, -1
do
    print(data[i])
end
```

数字 `for` 循环的格式是：

```lua
for 变量 = 初始值, 结束值, 步长 do
    -- 循环体
end
```

步长可以省略，默认值为 `1`。

## 6. 函数

Lua 中的函数是一种值，可以赋给变量，也可以保存到 `table` 中。

新建 `13.function.lua`。教程里按"先定义后调用 → 函数作为变量 → 参数数量 → 可变参数 → 多返回值"的顺序递进：

```lua
-- 脚本型语言，不能先调用再定义，因为代码是从上往下执行的
-- func1()

-- 第一种声明函数
function func1()
    print("这是func1")
end

-- 先定义，再调用，没有问题
func1()

-------------------------------------------------------

-- 将函数定义为一个变量——函数是一种数据类型
local func2 = function()
    print("这是func2")
end

func2()

-------------------------------------------------------

-- 带参数的函数
local func3 = function(a, b)
    print(a + b)
end

-- 函数的调用，实参可以多于形参，但少于形参时缺少的值为 nil
func3(5, 7)      -- 12
func3(5, 7, 9)   -- 12（第三个参数被忽略）

-------------------------------------------------------

-- 可变参数 ...
local func4 = function(...)
    -- 将不定数量的参数转换为table
    -- arg的作用域是func4函数体
    local arg = {...}

    local total = 0

    for k, v in pairs(arg)
    do
        total = total + v
    end

    print(total)
end

func4(1, 2, 3)       -- 6
func4(1, 2, 3, 4, 5) -- 15

-------------------------------------------------------

-- 多返回值
function func5()
    return 99, 100
end

-- 将多返回值，同时赋值给两个变量
local num1, num2 = func5()
print(num1)  -- 99
print(num2)  -- 100
```

## 7. 多脚本加载：require

随着代码增加，需要将不同功能拆分到多个文件中。Lua 可以使用 `require()` 加载模块。

`require()` 不只是简单地执行另一个脚本。它还会负责：

```text
根据模块名查找文件
-> 执行第一次加载到的模块
-> 保存模块返回值
-> 缓存加载结果
-> 后续加载同名模块时复用缓存
```

目录结构：

```text
project/
├── 15.require.lua
└── subfile/
    ├── config.lua
    └── subfile1.lua
```

### 7.1 查看模块搜索路径

`15.require.lua` 首先打印 `package.path`：

```lua
print(package.path)
```

`package.path` 是 Lua 查找普通 Lua 模块时使用的搜索规则。输出结果通常包含多个路径模板，例如：

```text
.\?.lua;.\?\init.lua
```

其中的 `?` 会被替换为传给 `require()` 的模块名。

例如：

```lua
require("subfile/subfile1")
```

Lua 会根据 `package.path` 中的模板尝试查找类似下面的文件：

```text
.\subfile\subfile1.lua
```

调用 `require()` 时通常不需要填写 `.lua` 扩展名，因为搜索模板中已经包含了 `.lua`。

需要注意，搜索路径通常相对于程序启动时的**当前工作目录**解析，不一定相对于正在执行的 `.lua` 文件。如果从不同目录启动程序，模块查找结果也可能不同。

### 7.2 加载并执行子文件

`subfile/subfile1.lua`：

```lua
print("subfile1")
```

`15.require.lua` 中第一次加载这个文件：

```lua
require("subfile/subfile1")
```

Lua 找到 `subfile/subfile1.lua` 后，会执行文件中的代码，因此控制台输出：

```text
subfile1
```

模块名更常见的写法是使用点号分隔层级：

```lua
require("subfile.subfile1")
```

在常见配置下，它也会查找：

```text
subfile/subfile1.lua
```

点号不是不能使用的特殊字符。对于模块层级，推荐优先使用 `subfile.subfile1` 这种写法。

### 7.3 使用 package.loaded 缓存模块

成功加载模块后，Lua 会将加载结果缓存在 `package.loaded` 中。键是传给 `require()` 的模块名：

```lua
require("subfile/subfile1")

-- 表会记录已经加载的文件记录
print(package.loaded["subfile/subfile1"])
```

`subfile1.lua` 没有主动返回值，因此 `require()` 会使用 `true` 表示模块已经成功加载。输出结果类似：

```text
subfile1
true
```

再次使用相同模块名加载时，**不会重复执行**。如果想验证"缓存机制确实在起作用"，可以主动清除缓存然后再次 `require`：

```lua
-- 先看缓存
print(package.loaded["subfile/subfile1"])  -- true

-- 手动清除缓存
package.loaded["subfile/subfile1"] = nil

-- 当每次加载文件时，都会检查package.loaded里面的内容，防止重复加载
-- package.loaded存储的是加载的路径字符串
-- 所以如果想重复多次加载一个文件，则需要再次加载前清除这个状态
require("subfile/subfile1")  -- 这次会重新执行 subfile1.lua！

-- 还可以用相对路径写法（不推荐）
-- "./"表示当前编写的lua文件所在的目录
require("./subfile/subfile1")

-- 获取另一个文件的局部变量
-- 子文件return的数据会返回给主文件的变量
local config = require("subfile/config")
print(config.appName)  -- hxsd
```

> 日常开发中，不建议随意清除模块缓存。这里只是演示 `package.loaded` 的缓存机制——模块初始化逻辑重复执行后，可能产生难以排查的问题。

### 7.4 模块名不同，缓存键也不同

课程示例中还使用了：

```lua
require("./subfile/subfile1")
```

这种写法不推荐使用。`require()` 接收的是**模块名**，不是普通文件路径。Lua 查找模块时，会将模块名中的点号转换为目录分隔符。因此，开头的 `.` 不应直接理解为文件路径中的“当前目录”。

如果确实使用不同写法加载模块，还要注意缓存键不同：

```lua
package.loaded["subfile/subfile1"]
package.loaded["./subfile/subfile1"]
```

对 `require()` 来说，这是两个不同的模块名。如果自定义搜索规则让它们最终指向同一个文件，它们仍然可能分别执行一次并产生两份缓存。

为了避免同一个文件被重复加载，项目中应该统一模块名写法。推荐写成：

```lua
require("subfile.subfile1")
```

不要在同一个项目中混用：

```lua
require("subfile.subfile1")
require("subfile/subfile1")
require("./subfile/subfile1")
```

### 7.5 使用 return 导出模块内容

如果子文件只有：

```lua
local appName = "hxsd"
```

其他脚本不能直接读取这个局部变量。模块可以创建一个 `table`，将需要公开的数据放入表中，再通过 `return` 返回。

`subfile/config.lua`：

```lua
local config = {}

config.appName = "hxsd"

return config
```

在 `15.require.lua` 中接收模块返回值：

```lua
local config = require("subfile.config")

print(config.appName) -- hxsd
```

执行流程：

```text
require("subfile.config")
-> 根据 package.path 找到 subfile/config.lua
-> 执行 config.lua
-> config.lua 返回 config 表
-> require() 将 config 表写入 package.loaded["subfile.config"]
-> 主文件使用局部变量 config 接收这个表
-> 通过 config.appName 读取公开数据
```

这种写法很适合用来组织配置和功能模块。模块内部数据优先使用 `local`，只通过返回的 `table` 公开外部真正需要访问的内容，可以减少全局变量被意外覆盖的问题。

### 7.6 完整示例

将教程 `15.require.lua` 的完整代码整理如下。它串联了搜索路径查看、缓存机制验证、相对路径、以及接收模块返回值：

```lua
-- 实现代码的切分

-- 预加载目录（查看模块搜索路径）
print(package.path)

print("-----------------------")

-- 将提供的lua文件中的代码执行一下
-- 文件名中包含特殊字符，比如"."，会导致加载错误
-- 文件扩展名".lua"不需要编写，因为会自动添加
require("subfile/subfile1")

-- 表会记录已经加载的文件记录
print(package.loaded["subfile/subfile1"])

-- 手动清除缓存
package.loaded["subfile/subfile1"] = nil

-- 当每次加载文件时，都会检查package.loaded里面的内容，防止重复加载
-- package.loaded存储的是加载的路径字符串
-- 所以如果想重复多次加载一个文件，则需要再次加载前清除这个状态
require("subfile/subfile1")

-- 加载文件的相对路径
-- "./"表示当前编写的lua文件，所在的目录
-- "../"表示当前编写的lua文件，所在的上级目录
require("./subfile/subfile1")

-- 获取另一个文件的局部变量
-- 子文件return的数据会返回给主文件的变量
local config = require("subfile/config")
print(config.appName)  -- hxsd
```

简单记：

| 内容                            | 作用                             |
| ------------------------------- | -------------------------------- |
| `require("subfile.config")`   | 查找、执行并缓存模块             |
| `package.path`                | 普通 Lua 模块的搜索规则          |
| `package.loaded`              | 已加载模块的缓存表               |
| `return config`               | 将模块希望公开的数据返回给调用者 |
| `local config = require(...)` | 接收模块导出的内容               |

## 8. 元表：metatable

Lua 的 `table` 可以保存数据和函数。除此之外，还可以为一个 `table` 绑定另一个 `table`，用来定义额外行为。这个用于扩展行为的表叫作**元表**（metatable）。

元表本身仍然是普通的 `table`。它的特殊之处在于，可以定义以双下划线开头的**元方法**（metamethod）。当 Lua 遇到对应操作时，会自动调用这些元方法。

例如：

| 元方法         | 触发场景                               |
| -------------- | -------------------------------------- |
| `__tostring` | 将值转换为字符串，例如调用 `print()` |
| `__add`      | 使用 `+` 运算符                      |
| `__index`    | 读取表中不存在的键                     |
| `__newindex` | 为表中不存在的键赋值                   |

这一节先使用 `__tostring`，让 `table` 打印出便于阅读的内容。

### 8.1 直接打印 Table

新建 `16.metatable.lua`。教程代码保留了完整的注释，并演示了两个表共享同一个元表的模式：

```lua
-- http://www.runoob.com/lua/lua-metatables.html
local t1 = { 1, 2, 3 }
local t2 = { 2, 3, 4 }

-- 直接打印表，显示的是内存地址
-- 期望打印table时，以人类易于阅读的结构显示 "{1, 2, 3}"，方便调试
print(t1)  -- table: 000001F...（难以阅读）

-- 打印时，就是将表作为字符串输出
-- 实现的功能：当需要将表作为字符串使用时，应该有一种办法
-- Lua提供了这种语法特性：metatable扩展（元表扩展）
local meta = {  -- 拥有扩展t1的元表，只要在元表中实现一些特殊的函数，
                -- 则t1就可以实现一些特殊功能，比如让t1作为字符串使用时触发自定义行为
    __tostring = function(tt) -- 元方法，当被扩展的表以string方式调用时触发
        local format = "{"

        for k, v in pairs(tt)
        do
            format = format .. v .. ', '
        end

        format = format .. "}"
        return format
    end
}

setmetatable(
    t1, -- 需要进行元表扩展的数据表
    meta
)

setmetatable(
    t2, -- 需要进行元表扩展的数据表
    meta    -- 两个表可以共享同一个元表
)

-- t1会被作为参数传递给元方法中的tt
print(t1)  -- {1, 2, 3, }
print(t2)  -- {2, 3, 4, }
```

运行结果类似：

```text
table: 000001F...
{1, 2, 3, }
{2, 3, 4, }
```

### 8.2 print(t1) 的完整调用流程

```text
print(t1)
↓
print() 需要将 t1 转换为字符串
↓
发现 t1 绑定了元表 meta
↓
找到 meta.__tostring
↓
调用 meta.__tostring(t1)        ← t1 传给形参 tt
↓
pairs(tt) 遍历 t1 中的数据
↓
返回 "{1, 2, 3, }"
↓
print() 输出字符串
```

> `__tostring` 中形参的名字可以自定义。课程脚本中使用 `tt`，无论叫什么，它接收到的都是当前需要转换为字符串的表。

### 8.3 多个 Table 共享同一个元表

从上面的完整示例可以看出，`meta` 同时绑定到了 `t1` 和 `t2`。每个 table 保存自己的数据，但共享元表中定义的行为规则。可以用 `getmetatable` 验证：

```lua
print(getmetatable(t1) == meta) -- true
print(getmetatable(t2) == meta) -- true
```

### 8.4 当前示例的限制

课程示例适合帮助理解元表，但还不是通用的 `table` 格式化工具。

需要注意：

- `pairs()` 不保证遍历顺序。示例输出通常符合数组顺序，但业务逻辑不应该依赖这一点。
- 示例只拼接 `value`，不会显示键。
- 如果值本身是嵌套 `table`，直接执行字符串拼接可能报错。
- 如果表中包含复杂值，需要判断类型并递归处理。
- 示例会在最后一个元素后面留下额外的逗号和空格。

对于元素连续排列的简单数组，可以使用 `table.concat()` 写出更简洁的版本：

```lua
local meta = {
    __tostring = function(target)
        return "{" .. table.concat(target, ", ") .. "}"
    end
}
```

运行结果：

```text
{1, 2, 3}
```

不过，`table.concat()` 也主要适用于连续数组，无法完整展示字符串键、不连续索引或嵌套 `table`。

### 8.5 延伸：使用 __add 自定义加法

元表不仅能修改字符串转换行为，还可以通过 `__add` 自定义 `+` 运算符。

例如，后面的练习题会让两个数组合并：

```lua
local result = data1 + data2
```

当 `data1` 绑定的元表中存在 `__add` 时，Lua 会将这段代码转换为类似下面的调用：

```lua
local result = meta.__add(data1, data2)
```

理解 `__tostring` 的调用流程后，再看 `__add` 会比较自然：都是在元表中定义规则，再由 Lua 在特定操作发生时自动调用。

## 9. 练习题

1. 编写函数，统计不规则 `table` 中键值对的总数。
2. 编写一个接收可变参数的函数，使用冒泡排序法排序，并返回参数个数和排序结果。
3. 使用元表的 `__add` 元方法，实现两个数组内容的合并。

### 9.1 统计不规则 Table 的键值对总数

不规则 `table` 可能包含字符串键、不连续的数字键或负数键，因此不能使用 `#data` 统计全部元素。可以使用 `pairs()` 遍历并主动计数：

```lua
local function countEntries(data)
    local count = 0

    for _ in pairs(data) do
        count = count + 1
    end

    return count
end

local data = {
    "aa",
    "bb",
    [4] = "dd",
    [-1] = "negative",
    name = "Lua"
}

print(countEntries(data)) -- 5
```

这里统计的是键值对总数，而不是数组长度。

### 9.2 使用冒泡排序处理可变参数

使用 `{ ... }` 可以将可变参数保存到数组中，再通过冒泡排序交换相邻元素：

```lua
local function bubbleSort(...)
    local data = { ... }
    local count = #data

    for i = 1, count - 1 do
        for j = 1, count - i do
            if data[j] > data[j + 1] then
                data[j], data[j + 1] = data[j + 1], data[j]
            end
        end
    end

    return count, data
end

local count, result = bubbleSort(5, 3, 8, 1, 2)

print(count)                  -- 5
print(table.concat(result, ", ")) -- 1, 2, 3, 5, 8
```

`data[j], data[j + 1] = data[j + 1], data[j]` 利用了 Lua 的多变量赋值，不需要额外声明临时变量。

### 9.3 使用 __add 合并两个数组

通过元表的 `__add` 元方法，可以自定义两个数组执行 `+` 运算时的行为：

```lua
local arrayMeta = {
    __add = function(left, right)
        local result = {}
-- 如果是连续数据，可以用ipairs，拼接后的表是按顺序的，
-- 如果不是连续数据，则要用pairs，不过拼起来的顺序无法保证
        for _, value in ipairs(left) do
            table.insert(result, value)
        end

        for _, value in ipairs(right) do
            table.insert(result, value)
        end

        return setmetatable(result, arrayMeta)
    end
}

local data1 = setmetatable({ 1, 2, 3 }, arrayMeta)
local data2 = setmetatable({ 4, 5, 6 }, arrayMeta)

local result = data1 + data2

print(table.concat(result, ", ")) -- 1, 2, 3, 4, 5, 6
```

执行 `data1 + data2` 时，Lua 会调用 `arrayMeta.__add(data1, data2)`。示例创建并返回一个新数组，因此不会修改原来的 `data1` 和 `data2`。
