# 背包道具信息弹窗实现

## 1. 文档目的

本文只记录“背包内点击道具后显示信息弹窗，再点击弹窗外部关闭”的实现方案。
内容基于某个 Unity 项目中的真实代码与预制体结构。

---

## 2. 代码入口

弹窗相关逻辑主要位于：

1. `Assets/Scripts/Features/UI/InventoryView.cs`
2. `Assets/Scripts/Features/UI/Components/InventoryGrid.cs`
3. `Assets/Scripts/Base/GameEvents.cs`
4. `Assets/Resources/UI/InventoryView.prefab`

### 2.1 `InventoryGrid` 的职责

格子脚本负责在点击时发出事件，不直接管理弹窗。

核心逻辑如下：

1. 格子在 `SetInfo` 中缓存当前道具 `currentItem`
2. 点击格子按钮时执行 `ShowInfoView`
3. `ShowInfoView` 中触发：

```csharp
GameEvents.RaiseInventoryItemClicked(currentItem);
```

也就是说，格子只负责把“哪个道具被点击了”这个事实抛给外部，不负责弹窗显示细节。

### 2.2 `GameEvents` 的职责

背包点击事件是一个全局静态事件：

```csharp
public static event Action<InventoryItem> OnInventoryItemClicked;
```

点击格子时通过：

```csharp
public static void RaiseInventoryItemClicked(InventoryItem inventoryItem)
{
    OnInventoryItemClicked?.Invoke(inventoryItem);
}
```

这一步的职责只是把道具数据往外传递。

### 2.3 `InventoryView` 的职责

`InventoryView` 监听 `OnInventoryItemClicked`，收到事件后：

1. 刷新信息面板显示内容
2. 打开弹窗层
3. 给遮罩绑定“点击关闭”的按钮事件

关键代码如下：

```csharp
public override void RegisterEventsOnStart()
{
    GameEvents.OnInventoryItemClicked += GameEvents_OnInventoryItemClicked;
}

private void GameEvents_OnInventoryItemClicked(InventoryItem obj)
{
    ShowItemInfo(obj);

    maskDisplay.GetComponent<Button>().onClick.RemoveAllListeners();
    maskDisplay.GetComponent<Button>().onClick.AddListener(() => { popUpRootDisplay.SetActive(false); });
}

private void ShowItemInfo(InventoryItem obj)
{
    popUpRootDisplay.SetActive(true);
    itemIconDisplay.sprite = obj.itemInfo.itemIcon;
    itemNameDisplay.text = obj.itemInfo.itemName;
    itemDescriptionDisplay.text = obj.itemInfo.itemDescription;
}
```

## 3. 预制体中的层级结构

根据 `InventoryView.prefab`，弹窗相关层级不是“挂在某个格子下面”，也不是简单的“挂在 InventoryBg 内部”，而是如下结构：

```text
InventoryView 根节点
- SafeArea
  - InventoryBg
    - 背包标题
    - 格子容器
    - 金币区域
    - 关闭按钮
- PopUpLayer
  - Mask
  - ContentRoot
    - itemIconDisplay
```

其中关键点如下：

### 3.1 `PopUpLayer` 是面板根层级的直接子节点

从预制体可见：

1. `PopUpLayer` 自身是全屏拉伸
2. `PopUpLayer` 的父节点不是 `InventoryBg`
3. `PopUpLayer` 的父节点也不是某个格子
4. `PopUpLayer` 和 `SafeArea` 是同级

这意味着：

1. 弹窗层不会继承某个格子的局部坐标系
2. 遮罩的全屏拉伸是真正相对整个 `InventoryView` 根节点
3. 弹窗不受背包内容布局影响

### 3.2 `popUpRootDisplay` 绑定的是 `PopUpLayer`

这是该实现中的一个关键细节。

在 `InventoryView.prefab` 中：

```text
popUpRootDisplay -> PopUpLayer
maskDisplay -> Mask
```

而不是：

```text
popUpRootDisplay -> ContentRoot
```

这意味着代码里：

```csharp
popUpRootDisplay.SetActive(false);
```

关闭的不是“信息面板内容本体”，而是**整个弹窗层**，包括：

1. 遮罩 `Mask`
2. 信息面板 `InfoPanel`

这套做法的好处是很直接：

1. 打开时，遮罩和面板一起出现
2. 关闭时，遮罩和面板一起消失
3. 不需要分别处理两个节点的显隐同步

## 4. 显示与关闭流程

### 4.1 初始化阶段

在 `InitInStart` 中：

```csharp
popUpRootDisplay.SetActive(false);
```

由于 `popUpRootDisplay` 指向 `PopUpLayer`，所以初始化时直接把整个弹窗层隐藏。

虽然预制体里 `PopUpLayer` 也默认是 `Inactive`，但代码中再次显式关闭是合理的，能保证运行时状态清晰一致。

### 4.2 点击格子阶段

流程如下：

1. 玩家点击某个背包格子
2. `InventoryGrid.ShowInfoView()` 触发 `GameEvents.RaiseInventoryItemClicked(currentItem)`
3. `InventoryView` 收到事件 `GameEvents_OnInventoryItemClicked`
4. `ShowItemInfo(obj)` 执行：
   1. `popUpRootDisplay.SetActive(true)`
   2. 刷新图标
   3. 刷新名称
   4. 刷新描述
5. 再给 `maskDisplay` 动态绑定点击关闭逻辑

### 4.3 点击遮罩关闭阶段

在每次打开弹窗时，都会这样处理：

```csharp
maskDisplay.onClick.RemoveAllListeners();
maskDisplay.onClick.AddListener(() => { popUpRootDisplay.SetActive(false); });
```

也就是说：

1. 先移除之前的按钮监听，避免重复绑定
2. 再重新绑定一次“点击后关闭整个 `PopUpLayer`”

最终效果就是：

1. 点击格子，`PopUpLayer` 打开
2. 点击 `Mask`，`PopUpLayer` 整体关闭

## 5 项目中的实际写法

该实现更偏向“预摆层级 + 整层显隐”的方案：

1. `PopUpLayer` 在预制体里提前摆好
2. `Mask` 和 `ContentRoot` 是同层级兄弟节点
3. `popUpRootDisplay` 直接指向 `PopUpLayer`
4. 打开和关闭都只控制整层

---

## 6. 如果小弹窗或遮罩挂在某个道具格子下面

只要小弹窗或遮罩挂在某个道具格子下面，就会出现：

1. 全屏拉伸只相对格子本地坐标系生效
2. 遮罩只能覆盖格子区域
3. 手动写死 `1080 x 1920` 也会因局部坐标和锚点关系产生偏移

而该方案之所以能稳定工作，原因是：

1. `PopUpLayer` 已经脱离格子层级
2. 它是 `InventoryView` 根节点下的全屏层
3. 遮罩 `Mask` 也是它下面的全屏子节点

所以“点击外部关闭”终于回到了正确的参考系。

## 7. 实现特点总结

### 7.1 弹窗不是运行时创建，而是预制体中预先存在

这意味着：

1. 层级稳定
2. 引用拖拽简单
3. 不需要额外查找节点

### 7.2 关闭的对象是整层，不只是面板内容

`popUpRootDisplay` 指向 `PopUpLayer`，所以关闭时：

1. 遮罩一起关
2. 信息面板一起关

这比只隐藏 `InfoPanel` 更直接。

### 7.3 弹窗刷新走的是事件通信

格子不直接依赖 `InventoryView`：

1. 格子发事件
2. `InventoryView` 响应事件
3. 由 `InventoryView` 决定如何显示弹窗

这种方式可以避免格子脚本直接持有复杂 UI 引用。
