### 主要窗口

Scene、Game、Hierarchy、Project、Inspector、Console，这六个窗口基本上就是Unity的门面了。

- **Scene**：场景编辑视图，在这里摆放物体、调整位置。
- **Game**：游戏运行时的摄像机视角，点击播放键之后看到的就是这里。
- **Hierarchy**：场景层级面板，列出了当前场景里所有的游戏物体，父子关系也在这里体现。
- **Project**：项目资源面板，所有的素材、脚本、预制体文件都在这里管理。
- **Inspector**：检查器，选中物体之后在这里查看和修改它的各种属性和组件。
- **Console**：控制台，报错、警告、`Debug.Log` 的输出都打在这里。

多用几次就都熟了，没什么别的捷径。

### Inspector 的 Normal 模式和 Debug 模式

Inspector 右上角有个小菜单，默认是 **Normal** 模式，切换成 **Debug** 模式之后会显示更多隐藏字段，包括私有变量也都露出来了。调试的时候挺有用的，可以直接看到脚本里那些没有 `[SerializeField]` 但又想瞄一眼的字段。

### Transform的父子关系

每个游戏物体默认都有 Transform 组件，记录着它的位置（Position）、旋转（Rotation）、缩放（Scale）。

父子级关系这里要注意一下：Inspector 里显示的是**局部坐标（Local）**，也就是相对于父物体的坐标。如果物体没有父级，那局部坐标和世界坐标是一样的。但一旦挂了父物体，子物体的 Position 就是相对于父物体原点的偏移量了。坐标和角度是加算，缩放是乘算的，父物体 Scale 是 (2,2,2)，子物体 Scale 是 (1,1,1)，子物体实际大小其实是 2 倍。有时候搞不清楚物体为什么跑偏了或者变大了，多半是父子级 Transform 没算清楚。

### GameObject 和 Component

Unity 的核心思想：**游戏物体（GameObject）** 本身什么都不是，它只是一个容器，功能全靠挂载的**组件（Component）** 来实现。

比如一个 Cube，它之所以能显示出来是因为有 `Mesh Renderer` 和 `Mesh Filter`，之所以有物理碰撞是因为有 `Collider`，之所以受重力是因为有 `Rigidbody`。你自己写的脚本挂上去之后也是一个 Component。这种组合式的设计挺好用的，想要什么功能加什么组件，不想要了删掉就行。

### 预制体 Prefab

Prefab 就是把一个配置好的 GameObject 保存成一个资源文件，之后可以反复实例化。改一处 Prefab，所有从它生成的实例都会跟着变，批量管理很方便。

几个操作要搞清楚：

- **Overwrite（Apply）**：把实例上的修改同步回 Prefab 本体，所有实例都会受影响。
- **Revert**：把实例上的修改撤销掉，恢复成 Prefab 的样子。
- **修改 Prefab 本体**：双击 Prefab 文件进入编辑模式，改完之后所有实例都更新。
- **删除 Prefab 文件**：场景里已经实例化的物体不会消失，但它们会变成"普通 GameObject"，和 Prefab 的关联断掉，Inspector 里原来蓝色的字会变成正常颜色。

导出 Prefab 的时候，记得勾选相关依赖（纹理、材质之类的），不然拿到别的项目里是一个粉色的缺材质物体，很丑。脚本如果有依赖其他脚本也要一并带上。

### Start 和 Update

写 Unity 脚本绕不开这两个方法：

```csharp
void Start()
{
    // 游戏开始时执行一次，做初始化用
}

void Update()
{
    // 每帧执行一次，游戏逻辑写这里
}
```

`Start` 只跑一次，适合做初始化，比如获取组件引用、设置初始值。`Update` 每帧都跑，帧率多高它就跑多快，所以涉及时间相关的逻辑要乘以 `Time.deltaTime`，不然在高帧率机器上物体会跑得飞快。

### Debug 类

调试的时候最常用的几个：

```csharp
Debug.Log("普通日志，白色");
Debug.LogWarning("警告，黄色");
Debug.LogError("报错，红色");
Debug.DrawLine(Vector3.zero, Vector3.up, Color.red); // 在 Scene 视图里画线，可视化调试用
Debug.DrawRay(transform.position, transform.forward * 5, Color.green); // 画射线
```

`DrawLine` 和 `DrawRay` 只在 Scene 视图里可见，Game 视图看不到，专门用来可视化一些方向、范围之类的东西，比打 Log 直观多了。

### Rotate 方法

`transform.Rotate()` 用来让物体旋转，有两种常见用法：

```csharp
// 以自身坐标系旋转（默认）
transform.Rotate(0, 90, 0);                          // 等价于下面这行
transform.Rotate(0, 90, 0, Space.Self);

// 以世界坐标系旋转
transform.Rotate(0, 90, 0, Space.World);
```

`Space.Self` 是沿着物体自己的轴旋转，`Space.World` 是沿着世界坐标的轴旋转。物体没有倾斜的时候两者看起来差不多，一旦物体本身被旋转过，区别就很明显了。

### 万向节死锁（Gimbal Lock）

这个是欧拉角的经典问题。当物体绕 X 轴旋转 90 度之后，Y 轴和 Z 轴会重合，这时候无论怎么转 Y 或者转 Z 效果都一样，少了一个自由度，这就是万向节死锁。

Unity 的 Transform 里显示的旋转是欧拉角，底层其实用的是四元数（Quaternion）来避免这个问题。所以在代码里做复杂旋转运算，最好直接用 `Quaternion`，别直接改 `eulerAngles`，不然容易踩坑。

### 类名和脚本名一致

Unity 里脚本文件名和里面的类名必须保持一致，不然挂不上去，Unity 找不到对应的类。起名字的时候直接想好再建，改起来麻烦，文件和类都要改。某些版本的Unity会解决这个问题，但是最好还是类名和文件名一致。

### RotateAround

```csharp
// 让物体绕某个点、某个轴旋转指定角度
transform.RotateAround(Vector3 point, Vector3 axis, float angle);

// 例如绕世界原点、Y轴旋转，模拟公转效果
transform.RotateAround(Vector3.zero, Vector3.up, 30 * Time.deltaTime);
```

`RotateAround` 是让物体**围绕某个点**转圈，同时自身的位置也会跟着变。用来做卫星绕行星转、摄像机绕目标旋转之类的效果很方便。

### RotateAround 和 Rotate 的区别

两个方法都可以传轴和角度，但效果完全不同：

|          | `Rotate(axis, angle)` | `RotateAround(point, axis, angle)` |
| -------- | ----------------------- | ------------------------------------ |
| 旋转中心 | 物体自身原点            | 指定的外部点                         |
| 位置变化 | 不变                    | 会跟着改变                           |
| 用途     | 自转                    | 公转                                 |

`Rotate` 只是自身原地转，`RotateAround` 会同时改变 `position` 和 `rotation`，因为物体要绕着外部的点移动。

### Vector2 和 Vector3 的方向值

几个常用的快捷方向向量，不用每次都手写坐标：

**Vector3：**

| 属性                | 值         |
| ------------------- | ---------- |
| `Vector3.up`      | (0, 1, 0)  |
| `Vector3.down`    | (0, -1, 0) |
| `Vector3.left`    | (-1, 0, 0) |
| `Vector3.right`   | (1, 0, 0)  |
| `Vector3.forward` | (0, 0, 1)  |
| `Vector3.back`    | (0, 0, -1) |
| `Vector3.zero`    | (0, 0, 0)  |
| `Vector3.one`     | (1, 1, 1)  |

**Vector2：**

| 属性              | 值      |
| ----------------- | ------- |
| `Vector2.up`    | (0, 1)  |
| `Vector2.down`  | (0, -1) |
| `Vector2.left`  | (-1, 0) |
| `Vector2.right` | (1, 0)  |
| `Vector2.zero`  | (0, 0)  |
| `Vector2.one`   | (1, 1)  |

`Vector3.forward` 对应 Z 轴正方向，Unity 里的"前方"是 Z 轴。
