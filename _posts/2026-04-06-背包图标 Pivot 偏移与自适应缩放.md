# 背包图标 Pivot 偏移与自适应缩放

## 1. 文档目的

本文只记录“背包格子中的图标显示”这一类问题，具体包括：

1. `Sprite Editor` 中调整 `pivot` 后，为什么 UI 中看起来没生效。
2. 为什么某些图标在背包格子中视觉上会偏。
3. 如何让图标在赋值时保持原图比例自适应缩放。
4. 如何在自适应缩放后，再按 `Sprite.pivot` 做视觉中心偏移。
5. 本次排坑过程中遇到的错误与最终方案。

---

## 2. 问题背景

背包格子中的每个图标，本质上是一个 `UI.Image`。需求分成两部分：

1. 图标不要变形，要根据原图宽高比例自适应缩放到格子预留区域内。
2. 某些图视觉中心不在几何中心，希望直接在 `Sprite Editor` 中通过修改 `pivot` 来矫正显示位置。

例如：

1. 正常图标使用 `pivot = 0.5, 0.5`
2. 某张图视觉中心偏左，于是切图时把它改成 `0.45, 0.5`
3. 希望赋值给背包图标组件时，这个视觉中心能自动体现在 UI 中

---

## 3. 为什么改了 Sprite.pivot，UI Image 看起来没有偏移

### 3.1 根因

这里最容易混淆的是两个不同的 `pivot`：

1. `Sprite.pivot`
2. `RectTransform.pivot`

#### `Sprite.pivot`

1. 属于图片资源本身
2. 单位是像素
3. 表示图片内部的参考点

#### `RectTransform.pivot`

1. 属于 UI 节点
2. 单位是 `0~1`
3. 表示这个 UI 矩形自身的轴心点

`UI.Image` 在显示时，不会自动使用 `Sprite.pivot` 去调整 `RectTransform` 的位置。
所以只在 `Sprite Editor` 里改中心点，并不会自动让背包图标发生期望中的偏移。

### 3.2 正确认知

如果想让切图时设置的视觉中心真正影响 UI，就必须在赋值 `sprite` 时：

1. 读取 `Sprite.pivot`
2. 把它换算成 `0~1` 的归一化值
3. 再转成 `anchoredPosition` 偏移量

---

## 4. 正确的参考系是什么

### 4.1 真实需求

希望实现的是：

1. 格子背景不动
2. 图标组件中心固定在格子中心
3. 图片的视觉中心来自 `Sprite.pivot`
4. 赋值时让“图片的视觉中心”对齐到“图标组件中心”

例如：

1. 图标组件中心是 `0.5`
2. 图片中心是 `0.45`
3. 结果应该是图片整体向右偏移 `0.05 * 实际显示宽度`

### 4.2 为什么之前会觉得公式不对

这是因为有两个不同的参考系：

#### 以图片自身为参考

若 `pivot.x = 0.45`，表示：

1. 从图片左边到 pivot 是 `45%`
2. 从 pivot 到图片右边是 `55%`

#### 以格子背景为参考

把这张图的 `pivot` 对齐到格子中心后，图片会整体向右挪。从格子角度看，就会表现为：

1. 图片离左边更远
2. 图片离右边更近

所以：

1. “图片内部左 45 右 55”是对的
2. “相对格子看图片向右偏”也是对的

这两句不冲突，只是参考系不同。

## 5. 本次踩过的坑

### 5.1 错误一：误用 `sprite.pivot.normalized`

曾尝试这样写：

```csharp
sprite.pivot.normalized
```

这不对。

原因：

1. `normalized` 是把向量按长度归一化到单位向量
2. 它不是“除以宽高后得到 0~1 的百分比”

如果想把 `Sprite.pivot` 变成 `0~1`，正确做法是：

```csharp
Vector2 normalizedPivot = new Vector2(
    sprite.pivot.x / sprite.rect.width,
    sprite.pivot.y / sprite.rect.height);
```

---

### 5.2 错误二：图标节点尺寸是负数

曾打印出日志：

```text
rectSize=(-30.00, -45.00), offset=(-1.50, 0.00), anchoredPosition=(-1.50, 0.00)
```

这说明不是 `pivot` 算错，而是用于计算偏移的图标显示尺寸本身就错了。

根因排查结果：

1. 把 `Icon` 锚点改回中心后
2. 发现预制体里它的宽高本身就是负数 `-30` 和 `-45`
3. 改成正常正数后，偏移结果恢复正常

### 5.3 负尺寸为什么会导致方向反转

偏移公式原本是：

```csharp
offsetX = (0.5f - normalizedPivot.x) * width
```

如果：

1. `normalizedPivot.x = 0.45`
2. `width = -30`

那结果就是：

```text
offsetX = (0.5 - 0.45) * -30 = -1.5
```

本来应该向右，结果被负宽度直接翻成了向左。

结论：

在计算 `pivot` 偏移之前，必须先保证图标节点尺寸是正确的正数。

## 6. 当前方案的总体思路

图标显示逻辑拆成两步：

1. 先按原图比例自适应缩放到图标可用区域内
2. 再根据 `Sprite.pivot` 计算视觉中心偏移

只有这样，偏移量才是基于“最终实际显示尺寸”算出来的，而不是基于原图像素尺寸瞎猜。

---

## 7. 自适应缩放规则

### 7.1 目标规则

需求是：

1. 如果原图偏宽，则宽度优先贴满，高度按比例缩放
2. 如果原图偏高，则高度优先贴满，宽度按比例缩放
3. 不允许变形拉伸

### 7.2 公式

设：

1. 图标组件预留的最大显示区域为 `maxSize`
2. 原图尺寸为 `sprite.rect.width` 和 `sprite.rect.height`

则：

```csharp
float widthScale = maxSize.x / sprite.rect.width;
float heightScale = maxSize.y / sprite.rect.height;
float scale = Mathf.Min(widthScale, heightScale);
```

含义：

1. `widthScale` 表示如果宽度贴满，需要放大或缩小多少倍
2. `heightScale` 表示如果高度贴满，需要放大或缩小多少倍
3. 取两者较小值，就能保证整张图都放得进区域里，不会溢出

最终显示尺寸为：

```csharp
Vector2 fittedSize = new Vector2(
    sprite.rect.width * scale,
    sprite.rect.height * scale);
```

这个结果正好符合需求：

1. 宽图优先贴宽
2. 高图优先贴高
3. 整体保持比例

## 8. Pivot 偏移公式

### 8.1 第一步：把 `Sprite.pivot` 转成 `0~1`

```csharp
Vector2 normalizedPivot = new Vector2(
    sprite.pivot.x / sprite.rect.width,
    sprite.pivot.y / sprite.rect.height);
```

例如：

1. 原图宽高 `100 x 100`
2. `pivot = (45, 50)`
3. 则 `normalizedPivot = (0.45, 0.5)`

### 8.2 第二步：拿组件中心 `0.5` 去对齐图片中心

图标组件中心固定看作 `0.5, 0.5`。

所以偏移量为：

```csharp
float offsetX = (0.5f - normalizedPivot.x) * fittedSize.x;
float offsetY = (0.5f - normalizedPivot.y) * fittedSize.y;
```

例子：

1. `normalizedPivot.x = 0.45`
2. `fittedSize.x = 100`

那么：

```text
offsetX = (0.5 - 0.45) * 100 = 5
```

也就是图片向右偏移 5。

### 8.3 为什么必须乘 `fittedSize`

因为图标真正显示出来的尺寸已经被缩放过了。
如果还拿原图像素尺寸去乘，那么偏移量就和屏幕上实际视觉尺寸对不上。

所以：

1. 自适应缩放先算 `fittedSize`
2. 再用 `fittedSize` 算 `pivot` 偏移

这是当前方案最关键的一点。

## 9. 当前脚本中的实现

当前 `InventoryGrid` 中的核心逻辑如下：

```csharp
public void SetInfo(InventoryItem inventoryItem)
{
    btnItem.onClick.RemoveAllListeners();

    currentItem = inventoryItem;
    itemIcon.color = new Color(1, 1, 1, 1);
    itemIcon.sprite = inventoryItem.itemInfo.itemIcon;

    ApplyAdaptiveSprite(inventoryItem.itemInfo.itemIcon);

    itemCount.text = "x " + inventoryItem.itemCount;
    btnItem.interactable = true;
    btnItem.onClick.AddListener(ShowInfoView);
}

private void ApplyAdaptiveSprite(Sprite sprite)
{
    RectTransform rectTransform = itemIcon.rectTransform;
    Vector2 maxSize = rectTransform.sizeDelta;

    float widthScale = maxSize.x / sprite.rect.width;
    float heightScale = maxSize.y / sprite.rect.height;
    float scale = Mathf.Min(widthScale, heightScale);

    Vector2 fittedSize = new Vector2(
        sprite.rect.width * scale,
        sprite.rect.height * scale);

    Vector2 normalizedPivot = new Vector2(
        sprite.pivot.x / sprite.rect.width,
        sprite.pivot.y / sprite.rect.height);

    float offsetX = (0.5f - normalizedPivot.x) * fittedSize.x;
    float offsetY = (0.5f - normalizedPivot.y) * fittedSize.y;

    rectTransform.sizeDelta = fittedSize;
    rectTransform.anchoredPosition = new Vector2(offsetX, offsetY);
}
```

## 11. 推荐的预制体配置

为了让这套方案稳定工作，建议图标节点满足：

1. 锚点用中心，不要用拉伸。
2. 宽高为明确正数。
3. `RectTransform.pivot` 保持 `0.5, 0.5`。
4. 位置偏移只通过 `anchoredPosition` 控制。

推荐理解是：

1. 图标组件本身定义“可用显示区域”
2. `Sprite.pivot` 定义“视觉中心”
3. 代码负责把“视觉中心”映射成最终偏移

---

## 10. 最终结论

这次图标显示问题的关键结论如下：

1. `Sprite.pivot` 不会自动影响 `UI.Image` 的显示位置。
2. 如果想让 `Sprite Editor` 中的视觉中心生效，必须在赋值时手动换算偏移。
3. `sprite.pivot.normalized` 是错误用法，必须自己除以 `sprite.rect.width/height`。
4. 偏移量必须基于最终显示尺寸 `fittedSize` 来算，而不是基于原图像素尺寸。
5. 图标节点的宽高必须为正数，否则偏移方向会被直接算反。
6. 当前实现已经能满足“图标自适应缩放 + 视觉中心微调”的目标。
